**¿Cuál es la diferencia entre DAC (Discretionary Access Control) y MAC (Mandatory Access Control)?**

<MultipleChoice>
  <Option>DAC usa contraseñas; MAC usa certificados digitales.</Option>
  <Option isCorrect>En DAC el propietario del recurso controla los permisos (como `chmod`). En MAC el sistema operativo impone políticas centralizadas que ni el propietario puede saltarse (como SELinux/AppArmor).</Option>
  <Option>DAC es para servidores; MAC es solo para escritorios.</Option>
</MultipleChoice>

**¿Qué hace este conjunto de reglas UFW?**

```bash
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
```

<MultipleChoice>
  <Option>Bloquea todo el tráfico de red del servidor.</Option>
  <Option isCorrect>Implementa una política restrictiva: bloquea todo el tráfico entrante por defecto, pero permite SSH, HTTP y HTTPS. El tráfico saliente está permitido.</Option>

## Ejercicio 7.5 — Caso práctico: hardening básico

Acabas de instalar un servidor nuevo accesible desde internet. Lista las 5 acciones de seguridad más importantes que realizarías inmediatamente:

<Aside type="note" title="Respuesta recomendada">

1. **Actualizar el sistema:**
   ```bash
   apt update && apt upgrade -y
   ```

2. **Configurar SSH seguro** (`/etc/ssh/sshd_config`):
   ```
   PermitRootLogin no
   PasswordAuthentication no
   Port 2222   # Cambiar puerto por defecto
   ```

3. **Activar el firewall UFW:**
   ```bash
   ufw default deny incoming
   ufw allow 2222/tcp  # nuevo puerto SSH
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw enable
   ```

4. **Crear usuario no-root con sudo** y deshabilitar el root:
   ```bash
   adduser adminuser
   usermod -aG sudo adminuser
   ```

5. **Instalar fail2ban** para bloquear IPs con demasiados intentos fallidos:
   ```bash
   apt install fail2ban -y
   systemctl enable --now fail2ban
   ```
</Aside>
