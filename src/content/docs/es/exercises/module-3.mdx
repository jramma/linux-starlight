## Ejercicio 3.1 — Gestión de usuarios

**¿Qué comando crea un nuevo usuario llamado `devuser` con su directorio home?**

<MultipleChoice>
  <Option>`useradd devuser`</Option>
  <Option isCorrect>`useradd -m devuser`</Option>
  <Option>`adduser --no-home devuser`</Option>
</MultipleChoice>

**¿Dónde se almacenan las contraseñas cifradas de los usuarios en Linux?**

<MultipleChoice>
  <Option>En `/etc/passwd` en texto cifrado.</Option>
  <Option isCorrect>En `/etc/shadow`, accesible solo por root.</Option>
  <Option>En `/home/.passwords` de cada usuario.</Option>
</MultipleChoice>

---

**¿Qué comando bloquea la cuenta del usuario `peter` sin eliminarla?**

<MultipleChoice>
  <Option>`userdel -l peter`</Option>
  <Option isCorrect>`usermod -L peter`</Option>
  <Option>`passwd --freeze peter`</Option>
</MultipleChoice>

## Ejercicio 3.2 — Grupos y sudo

**¿Cómo añades al usuario `ana` al grupo secundario `docker` sin quitarla de sus grupos actuales?**

<MultipleChoice>
  <Option>`usermod -g docker ana`</Option>
  <Option isCorrect>`usermod -aG docker ana`</Option>
  <Option>`groupadd docker ana`</Option>
</MultipleChoice>

---

**Analiza esta línea en `/etc/sudoers`:**

```
%developers ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx
```

**¿Qué permite esta regla?**

<MultipleChoice>
  <Option>Que cualquier usuario pueda reiniciar nginx sin contraseña.</Option>
  <Option isCorrect>Que todos los miembros del grupo `developers` puedan ejecutar `systemctl restart nginx` como root sin introducir contraseña.</Option>
  <Option>Que el usuario `developers` tenga acceso root completo sin contraseña.</Option>
</MultipleChoice>

<MultipleChoice>

## Ejercicio 3.5 — Caso práctico

Debes configurar un servidor para que:
1. El usuario `deploy` pueda reiniciar el servicio `nginx` con sudo sin contraseña
2. Nadie pueda conectarse por SSH con contraseña (solo claves)
3. El puerto SSH se cambie al 2222

**¿Cuáles son los pasos correctos?**

<Aside type="note" title="Respuesta">

**Paso 1** — Edita `/etc/sudoers` (con `visudo`):
```
deploy ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx
```

**Paso 2** — Edita `/etc/ssh/sshd_config`:
```
PasswordAuthentication no
PubkeyAuthentication yes
Port 2222
PermitRootLogin no
```

**Paso 3** — Recarga SSH:
```bash
sudo systemctl reload sshd
```

**¡Importante!** Antes de cerrar la sesión, abre una nueva conexión de prueba:
```bash
ssh -p 2222 deploy@servidor
```
</Aside>
