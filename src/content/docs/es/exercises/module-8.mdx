<MultipleChoice>

**Analiza este bloque de configuración de NGINX:**

```nginx
server {
    listen 80;
    server_name ejemplo.com www.ejemplo.com;
    root /var/www/ejemplo;
    index index.html index.php;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

**¿Qué hace la directiva `try_files $uri $uri/ =404`?**

<MultipleChoice>
  <Option>Redirige todas las peticiones a `/404.html`.</Option>
  <Option isCorrect>Intenta servir el archivo `$uri`, si no existe intenta `$uri/` (como directorio), y si tampoco existe devuelve un error 404.</Option>
  <Option>Comprime todos los archivos del sitio antes de servirlos.</Option>
</MultipleChoice>

## Ejercicio 8.4 — Preparación LFCS

**En el examen LFCS, recibes este escenario: "El servicio SSH del servidor no responde en el puerto 22". ¿Cuáles son los pasos metódicos para diagnosticarlo?**

<Aside type="note" title="Metodología de diagnóstico">

```bash
# 1. Verifica si el servicio está corriendo
systemctl status sshd

# 2. Comprueba si hay errores en los logs
journalctl -u sshd -n 30

# 3. Verifica si está escuchando en el puerto
ss -tlnp | grep :22

# 4. Comprueba el firewall
ufw status
iptables -L INPUT -n

# 5. Revisa la configuración
sshd -t   # Verifica la sintaxis de sshd_config

# 6. Si el servicio no arranca, reinícialo
sudo systemctl restart sshd
```
</Aside>

## Ejercicio 8.5 — Caso práctico final integrador

Tienes que desplegar un sitio web WordPress en un servidor Debian. Enumera los pasos en el orden correcto:

<Aside type="note" title="Solución completa">

```bash
# 1. Actualizar sistema
apt update && apt upgrade -y

# 2. Instalar la pila LEMP
apt install nginx mariadb-server php-fpm php-mysql -y

# 3. Asegurar MariaDB
mysql_secure_installation

# 4. Crear base de datos para WordPress
mysql -u root -p <<EOF
CREATE DATABASE wordpress;
GRANT ALL ON wordpress.* TO 'wpuser'@'localhost' IDENTIFIED BY 'contraseña_segura';
FLUSH PRIVILEGES;
EOF

# 5. Descargar WordPress
cd /var/www
wget https://wordpress.org/latest.tar.gz
tar -xzf latest.tar.gz
chown -R www-data:www-data wordpress/

# 6. Configurar NGINX para el sitio
# Crear /etc/nginx/sites-available/wordpress con el bloque server

# 7. Activar el sitio y recargar NGINX
ln -s /etc/nginx/sites-available/wordpress /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

# 8. Configurar el firewall
ufw allow 'Nginx Full'
ufw enable

# 9. Certificado SSL con Certbot
apt install certbot python3-certbot-nginx -y
certbot --nginx -d tudominio.com
```
</Aside>
