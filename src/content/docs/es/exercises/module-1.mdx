<Aside type="tip" title="Cómo usar esta práctica">
Lee cada paso **antes** de ejecutarlo. Si algo no funciona, revisa el paso anterior. Cuando termines, ejecuta el bloque de **Entrega** y lleva la salida al profesor.
</Aside>

---

## Objetivo

Al terminar esta práctica tendrás un servidor Ubuntu Server 24.04 corriendo y accesible por SSH desde tu ordenador, igual que un servidor remoto real. Elige el método que mejor se adapte a tu sistema.

---

## ¿Qué método usar?

| Método | Sistema | Requisitos | Complejidad |
|--------|---------|------------|-------------|
| **Multipass** | Windows y Linux | Hyper-V (Win) o KVM (Linux) | Baja |
| **WSL2** | Solo Windows | Windows 10/11 64-bit | Muy baja |
| **VirtualBox** | Windows, Mac y Linux | Cualquier sistema | Media |
| **KVM/virt-install** | Linux | KVM habilitado + ISO Ubuntu | Media-Alta |

<Aside type="note" title="Recomendación">
Si tienes Windows 10/11 o Linux moderno, empieza por **Multipass** — es la opción más rápida y no requiere descargar ISOs.
Usa **KVM/virt-install** si Multipass no funciona en tu distribución Linux o prefieres no usar snap.
</Aside>

## Paso 1 — Levantar el servidor

<Tabs>
<TabItem label="Multipass (Windows)">

Multipass es la herramienta oficial de Canonical para lanzar VMs de Ubuntu con un solo comando desde PowerShell.

**Requisitos:** Windows 10/11 con Hyper-V habilitado (viene activado por defecto en ediciones Pro/Enterprise; en Home puede activarse desde "Características de Windows").

### 1.1 Instalar Multipass

Abre **PowerShell como administrador** y ejecuta:

```powershell
winget install Canonical.Multipass
```

Cierra y vuelve a abrir PowerShell tras la instalación.

### 1.2 Lanzar el servidor

```powershell
multipass launch 24.04 --name servidor-linux --memory 2G --disk 20G
```

La primera vez descarga la imagen (~600 MB). Espera a que aparezca:

```
Launched: servidor-linux
```

### 1.3 Verificar que está corriendo

```powershell
multipass list
```

Deberías ver `servidor-linux` con estado `Running` y una IP local (por ejemplo `172.26.x.x`). Anota esa IP.

</TabItem>
<TabItem label="Multipass (Linux)">

Multipass también funciona en Linux usando KVM como backend.

**Requisitos:** KVM instalado y habilitado (la mayoría de distribuciones modernas lo incluyen).

### 1.1 Instalar Multipass

Abre una **terminal** y ejecuta:

```bash
sudo snap install multipass
```

Si no tienes snap:

```bash
# Debian/Ubuntu
sudo apt install snapd && sudo snap install multipass
```

### 1.2 Lanzar el servidor

```bash
multipass launch 24.04 --name servidor-linux --memory 2G --disk 20G
```

Espera a que aparezca:

```
Launched: servidor-linux
```

### 1.3 Verificar que está corriendo

```bash
multipass list
```

Anota la IP que aparece junto a `servidor-linux`.

</TabItem>
<TabItem label="WSL2 (solo Windows)">

WSL2 (Windows Subsystem for Linux) permite correr Ubuntu directamente en Windows sin VM ni hipervisor externo.

**Requisitos:** Windows 10 versión 2004 o posterior / Windows 11.

### 1.1 Instalar Ubuntu en WSL2

Abre **PowerShell como administrador**:

```powershell
wsl --install -d Ubuntu-24.04
```

Reinicia el ordenador si se te pide.

### 1.2 Configurar el usuario

Al arrancar Ubuntu por primera vez se te pedirá un nombre de usuario y contraseña. Usa:

- **Usuario:** `alumno`
- **Contraseña:** la que recuerdes (mínimo 8 caracteres)

### 1.3 Acceder al servidor

Desde PowerShell:

```powershell
wsl -d Ubuntu-24.04
```

O abre **Ubuntu 24.04** desde el menú Inicio.

<Aside type="note" title="Diferencia con los otros métodos">
Con WSL2 ya estás dentro del servidor directamente — no necesitas conectarte por SSH desde fuera. Los pasos de red del Paso 5 no aplican a WSL2.
</Aside>

</TabItem>
<TabItem label="VirtualBox">

Opción clásica, compatible con Windows, Mac y Linux.

**Requisitos:**
- [VirtualBox](https://www.virtualbox.org/wiki/Downloads) instalado
- ISO de Ubuntu Server 24.04 LTS ([descargar aquí](https://ubuntu.com/download/server)) — opción **Get Ubuntu Server**

### 1.1 Crear la máquina virtual

1. Abre VirtualBox y haz clic en **Nueva**
2. Rellena la configuración:
   - **Nombre:** `servidor-linux`
   - **Tipo:** Linux
   - **Versión:** Ubuntu (64-bit)
3. Haz clic en **Siguiente** en cada pantalla con estos valores:
   - **Memoria RAM:** 2048 MB (2 GB)
   - **Disco duro:** Crear uno nuevo → VDI → Reservado dinámicamente → **20 GB**
4. Haz clic en **Crear**

### 1.2 Configurar la red (dos adaptadores)

Con la VM seleccionada, ve a **Configuración → Red**:

- **Adaptador 1** → Conectado a: **NAT** (acceso a internet)
- **Adaptador 2** → ✅ Habilitar → Conectado a: **Adaptador solo-anfitrión** → `vboxnet0`

Si `vboxnet0` no existe, créalo en VirtualBox → Herramientas → Red → Crear.

### 1.3 Instalar Ubuntu Server

1. Selecciona la VM y haz clic en **Iniciar**; elige la ISO cuando la pida
2. En el instalador:
   - **Language:** English
   - **Keyboard:** Spanish → Spanish
   - **Type of install:** Ubuntu Server *(no la versión minimized)*
   - **Network:** deja DHCP automático
   - **Storage:** Use an entire disk → valores por defecto
   - **Profile setup:** nombre `Alumno Linux`, servidor `servidor-linux`, usuario `alumno`
   - **SSH:** ✅ Install OpenSSH server — **MUY IMPORTANTE**
   - **Featured snaps:** nada, clic en Done
3. Espera ~10-15 minutos y haz clic en **Reboot Now**

</TabItem>
<TabItem label="KVM/virt-install (Linux)">

Método nativo en Linux usando KVM + libvirt. No requiere VirtualBox ni snap. Necesitas tener la **ISO de Ubuntu Server 24.04** descargada ([descargar aquí](https://ubuntu.com/download/server)).

**Requisitos:** CPU con virtualización habilitada (Intel VT-x / AMD-V) y KVM disponible en el kernel.

### 1.1 Instalar dependencias

```bash
sudo apt install -y libvirt-clients libvirt-daemon-system virtinst
sudo systemctl enable --now libvirtd
sudo usermod -aG libvirt,kvm $USER
```

<Aside type="caution" title="Cierra sesión y vuelve a entrar">
El comando `usermod` no surte efecto hasta que cierres sesión completamente y vuelvas a entrar. Sin este paso los siguientes comandos darán error de permisos.
</Aside>

### 1.2 Preparar red y almacenamiento

libvirt gestiona sus propios ficheros en `/var/lib/libvirt/images/` — el usuario del hipervisor no puede acceder a tu carpeta personal.

```bash
# Iniciar la red virtual por defecto
sudo virsh net-start default
sudo virsh net-autostart default

# Copiar la ISO al almacenamiento de libvirt
sudo cp ~/Descargas/ubuntu-24.04.4-live-server-amd64.iso /var/lib/libvirt/images/

# Crear carpeta para el disco de la VM
sudo mkdir -p /var/lib/libvirt/images/servidor-linux
```

### 1.3 Lanzar el instalador

```bash
virt-install \
  --connect qemu:///system \
  --name servidor-linux \
  --memory 2048 \
  --vcpus 2 \
  --disk path=/var/lib/libvirt/images/servidor-linux/disk.qcow2,size=8,format=qcow2 \
  --location /var/lib/libvirt/images/ubuntu-24.04.4-live-server-amd64.iso,kernel=casper/vmlinuz,initrd=casper/initrd \
  --os-variant ubuntu24.04 \
  --network network=default \
  --graphics none \
  --console pty,target_type=serial \
  --extra-args "console=ttyS0,115200n8"
```

<Aside type="note" title="Por qué --location y no --cdrom">
Con `--cdrom` no es posible pasar argumentos al kernel (necesarios para redirigir la consola al puerto serie). Con `--location` apuntando al kernel y al initrd dentro de la ISO sí funciona.
</Aside>

### 1.4 Instalación interactiva

El instalador arrancará en tu terminal. Si aparece el menú:

```
Continue in rich mode
Continue in basic mode
View SSH instructions
```

Elige **"Continue in basic mode"** y sigue el asistente:

| Campo | Valor |
|-------|-------|
| Language | English |
| Keyboard | Spanish |
| Network | DHCP (Enter para confirmar) |
| Storage | Use an entire disk → Done |
| Your name | `Alumno Linux` |
| Server name | `servidor-linux` |
| Username | `alumno` |
| Password | la que recuerdes |
| Install OpenSSH | **Yes** — imprescindible |
| Featured snaps | ninguno, Done |

Espera ~10-15 minutos. Cuando aparezca **Reboot Now**, pulsa Enter.

</TabItem>
</Tabs>

## Paso 2 — Primera sesión en el servidor

<Tabs>
<TabItem label="Multipass (Windows y Linux)">

Entra en el servidor con:

```bash
multipass shell servidor-linux
```

</TabItem>
<TabItem label="WSL2">

Ya estás dentro si seguiste el paso anterior. Si cerraste la ventana, vuelve a abrir con:

```powershell
wsl -d Ubuntu-24.04
```

</TabItem>
<TabItem label="VirtualBox">

Cuando la VM arranque verás un prompt de login. Inicia sesión con `alumno` y tu contraseña.

</TabItem>
<TabItem label="KVM/virt-install (Linux)">

Tras el reinicio la VM arranca sola. Conéctate directamente por SSH (Paso 3) — no hay consola gráfica.

</TabItem>
</Tabs>

Ejecuta estos comandos para comprobar que todo funciona:

```bash
# Ver información del sistema
uname -a

# Ver las interfaces de red
ip addr show

# Actualizar la lista de paquetes
sudo apt update

# Instalar actualizaciones disponibles
sudo apt upgrade -y
```

<Aside type="caution" title="Si sudo pide contraseña">
Es normal la primera vez. Escribe tu contraseña (no se verán los caracteres mientras escribes).
</Aside>

## Paso 3 — Conectarte por SSH desde tu terminal

El objetivo es trabajar por SSH igual que con un servidor remoto real.

<Tabs>
<TabItem label="Multipass (Windows y Linux)">

### Obtener la IP del servidor

```bash
multipass info servidor-linux
```

Busca la línea `IPv4` — tendrás una IP del tipo `172.26.x.x` o `192.168.x.x`.

### Conectarte por SSH

Desde PowerShell (Windows) o terminal (Linux):

```bash
ssh ubuntu@<IP-del-servidor>
```

El usuario por defecto en Multipass es `ubuntu`. La primera vez preguntará si confías en el servidor — escribe `yes`.

<Aside type="tip" title="Sin contraseña">
Multipass configura autenticación por clave SSH automáticamente, no necesitas contraseña.
</Aside>

</TabItem>
<TabItem label="WSL2">

Con WSL2 no necesitas SSH — ya estás dentro del servidor. Si quieres practicar SSH de todas formas:

```powershell
# Desde PowerShell, obtener la IP de WSL
wsl hostname -I
```

Luego:

```powershell
ssh alumno@<IP-WSL>
```

</TabItem>
<TabItem label="VirtualBox">

### Encontrar la IP del adaptador Host-Only

Dentro de la VM ejecuta:

```bash
ip addr show
```

Busca la interfaz `enp0s8` (o `eth1`). Su IP empezará por `192.168.56.X`. Anótala.

Si no aparece, edita la configuración de red:

```bash
sudo nano /etc/netplan/00-installer-config.yaml
```

Asegúrate de que aparece `enp0s8`:

```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      dhcp4: true
```

Aplica los cambios:

```bash
sudo netplan apply
ip addr show enp0s8
```

### Conectarte por SSH

Desde PowerShell (Windows) o terminal (Mac/Linux):

```bash
ssh alumno@192.168.56.X
```

La primera vez escribe `yes` e introduce tu contraseña.

</TabItem>
<TabItem label="KVM/virt-install (Linux)">

### Obtener la IP del servidor

La VM recibe IP por DHCP en la red virtual de libvirt (`192.168.122.x`). Consulta las concesiones activas:

```bash
virsh --connect qemu:///system net-dhcp-leases default
```

Busca la línea con `servidor-linux` y anota su IP (será del tipo `192.168.122.X`).

### Conectarte por SSH

```bash
ssh alumno@192.168.122.X
```

La primera vez escribe `yes` e introduce tu contraseña.

<Aside type="tip" title="Si la VM no arranca tras el reinicio">
El instalador a veces apaga la VM en lugar de reiniciarla. Arráncala manualmente:

```bash
virsh --connect qemu:///system start servidor-linux
```
</Aside>

</TabItem>
</Tabs>

Si todo va bien, verás el prompt del servidor:

```
alumno@servidor-linux:~$
```

**Estás conectado por SSH a tu propio servidor Linux.**

## Paso 4 — Autenticación por clave SSH *(ejercicio extra)*

En lugar de escribir la contraseña cada vez que te conectas, puedes usar un par de claves SSH. Este método es el estándar en entornos profesionales.

### 4.1 Generar el par de claves en tu máquina

Si aún no tienes claves SSH en tu ordenador, créalas. Ejecuta esto **en tu máquina host** (PowerShell en Windows o terminal en Linux/Mac):

```bash
ssh-keygen -t ed25519 -C "alumno@clase"
```

Pulsa Enter tres veces para aceptar la ruta por defecto y sin passphrase. Se crearán dos ficheros:

- `~/.ssh/id_ed25519` — clave **privada** (nunca compartas este fichero)
- `~/.ssh/id_ed25519.pub` — clave **pública** (la que copiarás al servidor)

### 4.2 Copiar la clave pública al servidor

Sustituye la IP por la de tu servidor:

<Tabs>
<TabItem label="Linux / Mac / WSL2">

```bash
ssh-copy-id alumno@<IP-del-servidor>
```

Introduce la contraseña de `alumno` una última vez. A partir de ahora no volverá a pedirla.

</TabItem>
<TabItem label="Windows (PowerShell)">

En Windows moderno (10/11) `ssh-copy-id` no está disponible, pero puedes hacerlo manualmente:

```powershell
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh alumno@<IP-del-servidor> "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

Introduce la contraseña cuando la pida.

</TabItem>
</Tabs>

### 4.3 Verificar que funciona

```bash
ssh alumno@<IP-del-servidor>
```

Deberías entrar directamente **sin que te pida contraseña**.

<Aside type="note" title="¿Por qué es más seguro?">
Una clave privada de 256 bits es matemáticamente imposible de adivinar por fuerza bruta, al contrario que una contraseña corta. Además, si alguien roba tu contraseña no puede entrar; necesitaría también tu fichero de clave privada.
</Aside>

## Paso 5 — Configurar el hostname y la zona horaria

```bash
# Cambiar el hostname (nombre del servidor)
sudo hostnamectl set-hostname servidor-linux

# Configurar la zona horaria a España
sudo timedatectl set-timezone Europe/Madrid

# Verificar
timedatectl
```

## Bloque de entrega

Cuando termines todos los pasos, ejecuta este comando **desde la sesión SSH** (o desde dentro del servidor si usas WSL2):

```bash
echo "=== ENTREGA PRÁCTICA 1 ===" && \
echo "Hostname: $(hostname)" && \
echo "Usuario: $(whoami)" && \
uname -a && \
echo "--- Red ---" && \
ip addr show | grep "inet " && \
echo "--- Zona horaria ---" && \
timedatectl | grep "Time zone" && \
echo "--- Sistema actualizado ---" && \
sudo apt list --upgradable 2>/dev/null | wc -l && \
echo "paquetes pendientes (0 = actualizado)"
```

Copia toda la salida y llévala al profesor.

<Aside type="tip" title="Snapshot / backup antes de continuar">
**Multipass:**
```bash
multipass stop servidor-linux
multipass snapshot servidor-linux --name base-limpia-m1
multipass start servidor-linux
```
**VirtualBox:** Máquina → Tomar instantánea → "Base limpia M1"

**KVM/virt-install:**
```bash
virsh --connect qemu:///system snapshot-create-as servidor-linux base-limpia-m1
```
</Aside>
