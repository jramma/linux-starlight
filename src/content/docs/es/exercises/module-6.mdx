## Ejercicio 6.1 — Gestión de paquetes (apt)

**¿Qué diferencia hay entre `apt update` y `apt upgrade`?**

<MultipleChoice>
  <Option>Son sinónimos; ambos actualizan el sistema completo.</Option>
  <Option isCorrect>`apt update` refresca la lista de paquetes disponibles desde los repositorios. `apt upgrade` instala las versiones nuevas de los paquetes ya instalados.</Option>
  <Option>`apt update` actualiza el kernel; `apt upgrade` actualiza las aplicaciones.</Option>
</MultipleChoice>

---

**¿Cómo instalas el paquete `build-essential` sin que el sistema pida confirmación?**

<MultipleChoice>
  <Option>`apt install build-essential --confirm`</Option>
  <Option isCorrect>`apt install build-essential -y`</Option>
  <Option>`dpkg --install build-essential`</Option>
</MultipleChoice>

---

**¿Qué comando elimina un paquete y también sus archivos de configuración?**

<MultipleChoice>
  <Option>`apt remove nginx`</Option>
  <Option isCorrect>`apt purge nginx`</Option>
  <Option>`dpkg --delete-config nginx`</Option>
</MultipleChoice>

---

**¿Para qué sirve `apt autoremove`?**

<MultipleChoice>
  <Option>Elimina automáticamente los paquetes desactualizados.</Option>
  <Option isCorrect>Elimina paquetes que se instalaron como dependencias y ya no son necesarios porque el paquete principal fue desinstalado.</Option>
  <Option>Actualiza automáticamente todos los paquetes del sistema por la noche.</Option>
</MultipleChoice>

## Ejercicio 6.5 — Systemd Timers

**¿Cuál es la ventaja de usar un systemd timer frente a cron?**

<MultipleChoice>
  <Option>Los systemd timers son más rápidos porque usan memoria compartida.</Option>
  <Option isCorrect>Los timers integran logging con journalctl, pueden depender de otros servicios, soportan ejecución si el sistema estaba apagado (persistent), y permiten dependencias con systemd.</Option>
  <Option>Los timers de systemd permiten ejecutar scripts con mayor prioridad de CPU.</Option>
</MultipleChoice>

## Ejercicio 6.6 — Caso práctico

Necesitas programar un script de backup (`/usr/local/bin/backup.sh`) que se ejecute todos los días a las 3:00 AM. El script tarda unos 10 minutos y quieres que sus logs queden en journalctl.

**¿Cuál es la solución más robusta?**

<Aside type="note" title="Respuesta: systemd timer">

**1. Archivo de servicio** `/etc/systemd/system/backup.service`:
```ini
[Unit]
Description=Backup diario

[Service]
Type=oneshot
ExecStart=/usr/local/bin/backup.sh
```

**2. Archivo de timer** `/etc/systemd/system/backup.timer`:
```ini
[Unit]
Description=Timer para backup diario

[Timer]
OnCalendar=*-*-* 03:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

**3. Activar el timer:**
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now backup.timer
systemctl list-timers  # Verificar
```
</Aside>
