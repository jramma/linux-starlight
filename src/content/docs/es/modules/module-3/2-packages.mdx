Linux utiliza repositorios ("repos") para gestionar el software de forma segura. Generalmente no se descargan archivos `.exe` de la web.

<Tabs>
  <TabItem label="Debian / Ubuntu (apt)">
    Utiliza paquetes `.deb` (ver [[/es/modules/module-1/1-overview|Familias de Distribuciones]]).

    ```bash
    # 1. Actualizar la lista de paquetes disponibles (metadatos del repo)
    sudo apt update

    # 2. Instalar un paquete
    sudo apt install nginx

    # 3. Actualizar todos los paquetes instalados a la última versión
    sudo apt upgrade

    # 4. Eliminar un paquete
    sudo apt remove nginx

    # 5. Buscar un paquete
    apt search nginx
    ```
  </TabItem>

  <TabItem label="Fedora / RHEL (dnf)">
    Utiliza paquetes `.rpm`. `dnf` es el sucesor de `yum`.

    ```bash
    # 1. Verificar actualizaciones
    sudo dnf check-update

    # 2. Instalar (también actualiza los metadatos automáticamente)
    sudo dnf install nginx

    # 3. Actualizar todo el sistema
    sudo dnf update

    # 4. Eliminar un paquete
    sudo dnf remove nginx

    # 5. Ver información
    dnf info nginx
    ```

    <Aside type="note">
      En sistemas RHEL/CentOS 7 antiguos, verás `yum`. En RHEL 8/9, `yum` es solo un
      enlace simbólico a `dnf`.
    </Aside>
  </TabItem>

  <TabItem label="Arch (pacman)">
    Utiliza paquetes `.pkg.tar.zst` del repositorio oficial y el AUR (Arch User Repository).

    ```bash
    # 1. Sincronizar base de datos y actualizar el sistema completo
    sudo pacman -Syu

    # 2. Instalar un paquete
    sudo pacman -S nginx

    # 3. Actualizar todos los paquetes instalados
    sudo pacman -Su

    # 4. Eliminar un paquete (y sus dependencias huérfanas)
    sudo pacman -Rs nginx

    # 5. Buscar un paquete
    pacman -Ss nginx
    ```

    <Aside type="tip">
      En Arch, `pacman -Syu` siempre sincroniza la base de datos Y actualiza el sistema en un solo paso. Evita usar solo `-S` sin `-y` para no instalar versiones obsoletas.
    </Aside>
  </TabItem>
</Tabs>

## Ejercicios Prácticos

1. **Actualizar Primero**:
   - Ejecuta el comando de actualización apropiado para tu sistema (`apt update`, `dnf check-update` o `pacman -Syu`).
2. **Instalar Herramientas**:
   - Instala `git` y `curl`.
   - Verifica que están instalados comprobando sus versiones (`git --version`).
3. **Buscar**:
   - Busca un paquete llamado `htop`.
   - Instálalo y ejecútalo.
4. **Limpiar**:
   - Elimina `htop` si no lo quieres.
