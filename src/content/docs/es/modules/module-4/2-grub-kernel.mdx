## 1. Configurar GRUB de forma permanente

Si quieres que tu servidor siempre arranque enseñando más mensajes de depuración en pantalla, o que la cuenta atrás de encendido pase de 5 segundos a 1, no puedes editar la pantalla negra que sale al encender. Tienes que arrancar Debian, entrar a tus archivos y editar el fichero maestro de GRUB.

```bash
sudo nano /etc/default/grub
```

Dentro, te toparás con un par de variables importantes:

- `GRUB_TIMEOUT=5`: El tiempo en segundos que el servidor espera en la pantalla del menú antes de elegir la primera opción automáticamente y arrancar.
- `GRUB_CMDLINE_LINUX_DEFAULT="quiet"`: Los parámetros que se le pasan al Kernel. `quiet` silencia todo para que el arranque sea estético. Si un día tu sistema falla y necesitas ver en qué línea exacta de texto ocurre el fallo de carga, deberías **borrar** la palabra `quiet`.

<Aside type="caution" title="La regla de oro de GRUB">
  Editar `/etc/default/grub` **NO** aplica los cambios. Ese fichero es solo una
  plantilla base pensada para humanos. Para que el Gestor de arranque real del
  disco duro recoja los cambios, tras guardar el archivo siempre debes ejecutar
  religiosamente el comando: `sudo update-grub`
</Aside>

## 2. Salvar el Servidor (Romper GRUB en caliente)

Imagina este escenario de examen LFCS o vida real: Heredas un servidor físico de un antiguo empleado que fue despedido. Nadie sabe la contraseña de Root ni de ningún usuario. Estás totalmente bloqueado.
En Linux, si tienes acceso físico al teclado del servidor, **puedes hackear tu propio administrador rompiendo el proceso de arranque**.

1. Reinicias el PC. Cuando salga la pantalla de GRUB listando a Debian, en lugar de darle a `Enter`, pulsas la tecla minúscula **`e` (Edit)**.
2. Esto abre en la pantalla pre-encendido un editor rudimentario. Debes buscar la línea larga de texto que empieza por la palabra `linux` o `linuxefi`.
3. Te vas al absoluto **final de esa línea** usando las flechas direccionales del teclado, pones un espacio, y escribes esta poción mágica:
   `init=/bin/bash`
4. Presionas `Ctrl + X` para arrancar.

**¿Qué acabas de hacer?**
Has interrumpido la Fase 5 del arranque normal. Le has dicho al Kernel: _"Ignora a Systemd y no me pidas ninguna contraseña. En lugar de eso, inyectame directamente a tu cerebro una terminal desnuda de tipo bash"_.
Aparecerá un cursor visual. Ejecutas el comando `passwd`, escribes tu cuenta nueva de administrador, y reinicias. Has recuperado la empresa.

## 3. Módulos del Kernel (Drivers)

En Windows se instalan "Drivers". En Linux, el Kernel es tan gigantesco que todos los drivers del mundo, desde antenas Wi-Fi hasta joysticks, ya vienen dentro de él o se bajan del repositorio.
Sin embargo, para no consumir toda la RAM del servidor encendiendo un millón de drivers que no usas, estos se cargan y descargan "Módulos dinámicos".

### Ver lo que está encendido (`lsmod`)

Listar Modules. Muestra una tabla con todos los drivers que el Kernel está usando ahora mismo en RAM.

```bash
lsmod
```

### Inyectar un Modulo (`modprobe`)

Si conectas un dispositivo extraño por USB y Linux no lo pilla, puedes forzar al Kernel a inyectarse ese driver en nombre y alma a su RAM.

```bash
# Force-load driver genérico de Bluetooth
sudo modprobe bluetooth
```

_(Si quisieras quitarle el módulo, como quien le saca un cable, usarías `modprobe -r`)_.

### Obtener información de un módulo (`modinfo`)

Antes de cargar un módulo desconocido puedes inspeccionar su ficha técnica: descripción, autor, versión, parámetros que acepta y dependencias con otros módulos.

```bash
modinfo e1000
# name:        e1000
# description: Intel(R) PRO/1000 Network Driver
# author:      Intel Corporation ...
# depends:     (lista de módulos requeridos)
# parm:        InterruptThrottleRate: ...
```

### Carga permanente al arrancar (`/etc/modules-load.d/`)

`modprobe` carga el módulo solo hasta el próximo reinicio. Para que un módulo se inyecte automáticamente en cada arranque, crea un fichero en `/etc/modules-load.d/`:

```bash
# Hacer permanente la carga del driver e1000
echo "e1000" | sudo tee /etc/modules-load.d/e1000.conf

# Verificar que systemd lo procesará en el próximo arranque
cat /etc/modules-load.d/e1000.conf
```

<Aside type="tip" title="Truco LFCS">
El fichero puede tener cualquier nombre con extensión `.conf`. Cada línea es el nombre de un módulo. Los comentarios se inician con `#`. El servicio `systemd-modules-load.service` los procesa durante la Fase 5 del arranque.
</Aside>
