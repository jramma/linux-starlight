Instal·lar Linux en un servidor és diferent de fer-ho en un ordinador d'escriptori. Els servidors se solen executar **sense interfície gràfica** (mode headless), estar optimitzats per a estabilitat i seguretat, i gestionar-se remotament a través d'**SSH**.

## Quin tipus de servidor tens?

### Servidor físic (bare metal)

Un ordinador dedicat o una màquina a la teva xarxa local. S'instal·la Linux directament des d'un USB booteable, igual que en un PC, però amb variants específiques per a servidor.

### VPS (Virtual Private Server)

Un servidor virtual llogat a un proveïdor de núvol (Hetzner, DigitalOcean, OVH, etc.). Normalment el proveïdor t'ofereix un panell web on pots seleccionar el sistema operatiu i instal·lar-lo amb un clic. No necessites USB.

### Servidor dedicat remot

Igual que un servidor físic però accessible remotament. Molts proveïdors ofereixen **IPMI/iDRAC/iLO** per a accés a la consola com si estiguessis físicament al davant.

## Distros recomanades per a servidor

Per a un entorn de servidor lo habitual és triar distribucions **LTS** orientades a estabilitat:

| Distribució | Empresa | Cicle LTS | Ús típic |
|-------------|---------|-----------|------------|
| Ubuntu Server | Canonical | 5 anys | Web, desenvolupament, cloud |
| Debian | Comunitat | ~5 anys | Estable, minimalista |
| Rocky Linux | RESF | 10 anys | Empresarial (substitut de RHEL) |
| AlmaLinux | AlmaLinux OS | 10 anys | Empresarial (substitut de RHEL) |
| Fedora Server | Red Hat | ~1 any | Avantguarda tecnològica |

> Per a la majoria de casos, **Ubuntu Server LTS** o **Debian** són les opcions més documentades i amb major comunitat.

## Instal·lació en servidor físic

### Pas 1 — Descarrega la ISO de servidor

Descarrega la versió **Server** de la teva distribució, no la d'escriptori. Per exemple:

- [Ubuntu Server 24.04 LTS](https://ubuntu.com/download/server)
- [Debian Netinstall](https://www.debian.org/distrib/netinst)

La versió server és més lleugera (no inclou entorn gràfic) i té opcions específiques per a servidors.

### Pas 2 — Crea l'USB booteable

Segueix els passos de [Crear un USB Booteable](./bootable-usb) usant Rufus, Etcher o `dd`.

### Pas 3 — Instal·lació en mode text

Els instal·ladors de servidor solen funcionar en mode text (sense entorn gràfic). No t'amoïnis, és molt guiat:

**Ubuntu Server:**
1. Selecciona l'idioma
2. Configura la xarxa (IP estàtica o DHCP)
3. Configura el disc:
   - **Usar disc complet** (recomanat)
   - O **personalitzat** per a LVM, RAID, etc.
4. Configura l'usuari i la contrasenya
5. Activa l'opció **Install OpenSSH server** ✅ (¡important per a accés remot!)
6. Selecciona snaps addicionals si es desitja (Docker, etc.)
7. Completa la instal·lació

### Pas 4 — Configuració post-instal·lació

Un cop instal·lat, connecta't per SSH des de Windows:

```powershell
# Des de PowerShell o CMD a Windows 10/11
ssh usuari@IP_DEL_SERVIDOR
```

O usa [PuTTY](https://putty.org) si prefereixes una interfície gràfica.

## Instal·lació en VPS (núvol)

### Mètode 1 — Panell de control del proveïdor

La majoria de proveïdors ofereixen instal·lació amb un clic des del seu panell:

1. Accedeix al panell de control del VPS
2. Ves a **Sistema operatiu** o **Reinstal·lar**
3. Selecciona la distribució i versió (p. ex., Ubuntu 24.04 LTS)
4. Confirma la instal·lació. El proveïdor instal·larà Linux automàticament
5. Rebràs les credencials d'accés per email o les configuraràs amb la teva clau SSH

### Mètode 2 — Autenticació amb clau SSH (recomanat)

En lloc de contrasenyes, els servidors moderns usen **claus SSH** per a major seguretat.

**Generar un parell de claus a Windows (PowerShell):**
```powershell
ssh-keygen -t ed25519 -C "mi@correo.com"
```

Això crea:
- `~/.ssh/id_ed25519` → clau privada (¡mai la comparteixis!)
- `~/.ssh/id_ed25519.pub` → clau pública (la puges al servidor)

**Afegir la clau pública al VPS:**
```bash
# Des del servidor (primera vegada amb contrasenya)
mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys
# Enganxa el contingut de id_ed25519.pub
```

Un cop configurat, connectaràs sense contrasenya:
```powershell
ssh -i ~/.ssh/id_ed25519 usuari@IP_SERVIDOR
```

## GRUB en servidors

El gestor d'arrencada en servidors funciona igual que en escriptori, però amb algunes particularitats:

### Configuració típica de GRUB en servidor

```bash
# Veure configuració actual
cat /etc/default/grub

# Línies rellevants per a servidor:
GRUB_TIMEOUT=5                    # Temps d'espera abans d'arrencar
GRUB_CMDLINE_LINUX_DEFAULT="quiet"  # Sense splash en servidor
GRUB_TERMINAL=console             # Només consola, sense gràfics
```

### Actualitzar GRUB després de canvis

```bash
# Debian/Ubuntu
sudo update-grub

# Rocky Linux / AlmaLinux / Fedora
sudo grub2-mkconfig -o /boot/grub2/grub.cfg
```

### GRUB en sistemes UEFI amb servidor

En servidors moderns amb UEFI, GRUB s'instal·la a la partició EFI:

```bash
# Verificar que GRUB està a la partició EFI
efibootmgr -v

# Reinstal·lar GRUB si cal (Ubuntu/Debian)
sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB
sudo update-grub
```

## Configuració bàsica post-instal·lació

### Actualitzar el sistema

```bash
# Ubuntu/Debian
sudo apt update && sudo apt upgrade -y

# Rocky Linux / AlmaLinux
sudo dnf update -y
```

### Configurar el tallafocs

```bash
# UFW (Ubuntu/Debian)
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
sudo ufw status

# Firewalld (Rocky/AlmaLinux/Fedora)
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### Desactivar accés SSH amb contrasenya (un cop configurades les claus)

```bash
sudo nano /etc/ssh/sshd_config
```

Canvia o afegeix aquestes línies:
```
PasswordAuthentication no
PermitRootLogin no
PubkeyAuthentication yes
```

Reinicia SSH:
```bash
sudo systemctl restart sshd
```

### Configurar zona horària

```bash
sudo timedatectl set-timezone Europe/Madrid
timedatectl status
```

## Verificar el servidor

```bash
# Informació del sistema
hostnamectl

# Ús de recursos
top
htop  # Si està instal·lat: sudo apt install htop

# Espai en disc
df -h

# Ports oberts
ss -tulnp

# Logs del sistema
sudo journalctl -xe
```
