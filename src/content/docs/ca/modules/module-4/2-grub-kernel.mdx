## 1. Configurar GRUB de forma permanent

Si vols que el teu servidor sempre arrenqui ensenyant més missatges de depuració a la pantalla, o que el compte enrere d'engegada passi de 5 segons a 1, no pots editar la pantalla negra que surt en engegar. Has d'arrencar Debian, entrar als teus fitxers i editar el fitxer mestre de GRUB.

```bash
sudo nano /etc/default/grub
```

A dins, et trobaràs amb un parell de variables importants:

- `GRUB_TIMEOUT=5`: El temps en segons que el servidor espera a la pantalla del menú abans de triar la primera opció automàticament i arrencar.
- `GRUB_CMDLINE_LINUX_DEFAULT="quiet"`: Els paràmetres que se li passen al Kernel. `quiet` silencia tot perquè l'arrencada sigui estètica. Si un dia el teu sistema falla i necessites veure en quina línia exacta de text ocorre la fallada de càrrega, hauries d'**esborrar** la paraula `quiet`.

<Aside type="caution" title="La regla d'or de GRUB">
  Editar `/etc/default/grub` **NO** aplica els canvis. Aquest fitxer és només una
  plantilla base pensada per a humans. Perquè el Gestor d'arrencada real del
  disc dur reculli els canvis, després de desar el fitxer sempre has d'executar
  religiosament el comandament: `sudo update-grub`
</Aside>

## 2. Salvar el Servidor (Trencar GRUB en calent)

Imagina aquest escenari d'examen LFCS o vida real: Heretes un servidor físic d'un antic empleat que va ser acomiadat. Ningú sap la contrasenya de Root ni de cap usuari. Estàs totalment bloquejat.
A Linux, si tens accés físic al teclat del servidor, **pots hackejar el teu propi administrador trencant el procés d'arrencada**.

1. Reinicies el PC. Quan surti la pantalla de GRUB llistant Debian, en lloc de prémer `Enter`, polses la tecla minúscula **`e` (Edit)**.
2. Això obre a la pantalla pre-engegada un editor rudimentari. Has de cercar la línia llarga de text que comença per la paraula `linux` o `linuxefi`.
3. Te'n vas a l'absolut **final d'aquesta línia** usant les fletxes direccionals del teclat, poses un espai, i escrius aquesta poció màgica:
   `init=/bin/bash`
4. Pressiones `Ctrl + X` per arrencar.

**¿Què acabes de fer?**
Has interromput la Fase 5 de l'arrencada normal. Li has dit al Kernel: _"Ignora Systemd i no em demanis cap contrasenya. En lloc d'això, injecta'm directament al teu cervell una terminal nua de tipus bash"_.
Apareixerà un cursor visual. Executes el comandament `passwd`, escrius el teu compte nou d'administrador, i reinicies. Has recuperat l'empresa.

## 3. Mòduls del Kernel (Drivers)

A Windows s'instal·len "Drivers". A Linux, el Kernel és tan gegantí que tots els drivers del món, des d'antenes Wi-Fi fins a joysticks, ja venen a dins seu o es baixen del repositori.
Tanmateix, per no consumir tota la RAM del servidor encenent un milió de drivers que no uses, aquests es carreguen i descarreguen com a "Mòduls dinàmics".

### Veure el que està encès (`lsmod`)

Llistar Mòduls. Mostra una taula amb tots els drivers que el Kernel està usant ara mateix a la RAM.

```bash
lsmod
```

### Injectar un Mòdul (`modprobe`)

Si connectes un dispositiu estrany per USB i Linux no el detecta, pots forçar el Kernel a injectar-se aquest driver en nom i ànima a la seva RAM.

```bash
# Force-load driver genèric de Bluetooth
sudo modprobe bluetooth
```

_(Si volguessis treure'n el mòdul, com qui en treu un cable, usaries `modprobe -r`)_.
