Hauràs notat que en engegar Linux, la xarxa funciona, el servidor SSH et deixa entrar, i el servidor Web funciona. Però tu no vas iniciar aquests programes a mà executant aplicacions.

Aquests programes s'anomenen **Dimonis** (Daemons, que se sol abreujar amb una 'd' al final, com `sshd` o `systemd`). Són programes invisibles i sense interfície que viuen de fons a la memòria RAM. Al capatàs total de tots aquests programes que decideix qui neix, qui mor i qui reinicia se l'anomena **Systemd**.

I a menys que instal·lis Linux des de 0 al teu smartwatch, usaràs el seu comandament principal: `systemctl`.

---

## 1. El Control de Comandament: `systemctl`

El comandament Control de Systemd (`systemctl`) espera una ordre i el nom d'una Unitat (generalment finalitzada en `.service`).
Usem d'exemple el servidor web de Debian (`nginx` o `apache2`).

### Veure la Salut del Sistema (Status)

```bash
systemctl status nginx
```

Això t'imprimirà un quadre vital inestimable. D'una sola ullada veuràs:

- Si està en verd (**active (running)**) o vermell (**failed**).
- Si ha col·lapsat, imprimirà textualment les 10 últimes línies del fitxer d'error explicant-te _per què_.

### Aturar i Engegar (Start / Stop)

Per fer la modificació en calent de la memòria RAM.

```bash
sudo systemctl stop nginx
sudo systemctl start nginx
```

### L'Art de Reiniciar vs Recarregar

Si canvies un fitxer de configuració de Nginx (per exemple, afegeixes una pàgina web nova), el Dimoni que resideix a la memòria RAM no se n'assabenta perquè no està llegint el fitxer de text a cada segon. Has de forçar que el llegeixi. Tens dos enfocaments:

- **La Ruleta Russa (`restart`)**:

  ```bash
  sudo systemctl restart nginx
  ```

  Això mata instantàniament el procés, talla la connexió de tots els clients que s'estan descarregant coses de la teva web, i engega un procés nou. És una animalada en Producció si no hi ha un error greu.

- **El Toc Cuidat (`reload`)**:
  ```bash
  sudo systemctl reload nginx
  ```
  Li envia una senyal elegant al Dimoni: _"Ei, quan acabis d'atendre les descàrregues que tens a mitges, rellegeix el fitxer de configuració a veure si hi ha hagut canvis"_. Zero micro-talls, màxima professionalitat.

## 2. Immortalitzar un Servei (Enable / Disable)

Que tu facis `start` a un servei web ara mateix, no assegura de cap manera que si el servidor pateix un tall elèctric i reinicia, el servei s'hagi d'auto-arrencar.

Per enganxar profundament un Dimoni al cicle d'engegada de la Fase 5 del Sistema, i garantir la seva execució automàtica després de reiniciar, l'hem d'**Habilitar**:

```bash
# Enganchar l'arrencada automàtica ("Crear Symlink")
sudo systemctl enable nginx

# Retirar-lo de l'arrencada automàtica (Eliminar Symlink)
sudo systemctl disable nginx

# Si vols que s'engegui ARA MATEIX i a més enganxar-lo al futur
# (Truc LFCS amb el modificador --now)
sudo systemctl enable --now nginx
```
