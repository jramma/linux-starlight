## 1. El Primer Inici de Sessió i Usuaris

Quan arrencarà Debian per primera vegada, seràs rebut pel prompt (indicador) d'inici de sessió:

```bash
debian-server login:
```

### L'Usuari Root (Administrador)

Durant la instal·lació de Debian, et va preguntar la contrasenya per a `root`.

- `root` és el superusuari absolut. No té restriccions. Pot esborrar el nucli del sistema operatiu en un mil·lisegon si li ho ordenes.
- **Regla d'Or en Producció:** MAI permetre inicis de sessió remots directes (per SSH) usant `root`, i és preferible no iniciar sessió localment com a ell, tret que el sistema estigui trencat.

### Usuaris Regulars i Sudo

Per seguretat, accedim amb l'usuari regular que vam crear en la instal·lació (ex. `joan`). Per a tasques que requereixen tocar `/etc` o gestionar la xarxa, «elevem» els nostres privilegis temporalment usant `sudo` (Super User DO).

```bash
# Ets 'joan'. Vols actualitzar l'índex de paquets:
sudo apt update
```

Si per accident Debian no va instal·lar `sudo` o no et va afegir al seu grup (`sudo` a Debian, `wheel` a RHEL), la forma clàssica Unix d'elevar privilegis és:

```bash
# Canvia el teu entorn al de root (et demanarà la clau de root, no la teva)
su -
```

## 2. Identitat a la Xarxa: Hostname

És crític en una granja de servidors saber exactament en quin estàs treballant. El _hostname_ és el nom de la màquina.

```bash
# Veure informació del sistema operatiu, versió del nucli i hostname
hostnamectl status

# Reanomenar permanentment la màquina (requereix sudo)
sudo hostnamectl set-hostname db-server-01
```

Aquest canvi es desa subreptíciament a `/etc/hostname`. Alguns serveis de xarxa (i bases de dades locals) exigeixen poder resoldre aquest nom localment, de manera que també és bona pràctica afegir o editar la línia corresponent a `/etc/hosts`:

```
127.0.1.1       db-server-01
```

## 3. Preparació per a Accés Remot (SSH)

Mai administraràs un servidor assegut en una cadira de la freda sala de servidors connectat al seu monitor directament. Tot el manteniment global es fa mitjançant **SSH (Secure Shell)**.

### El _Daemon_ SSH

Debian instal·la `openssh-server` si el vas marcar durant la instal·lació de paquets. A nivell de procés, s'anomena _Dimoni_ (Daemon) de SSHD.

```bash
# Verificar si el servei SSH està en funcionament
systemctl status ssh
```

<Aside type="caution" title="Comandes `systemctl` i Systemd">
  Avui no aprofundirem, però recorda del mòdul anterior que `systemd`
  gestiona si un programa s'encén amb la màquina o no. Aprendrem això a
  fons a la **Classe 3**. Recorda la comanda superior!
</Aside>

### Connectant-se Remotament

Des del teu propi portàtil (ja sigui MacOS, Windows amb PowerShell o el teu Linux Desktop d'ús diari), obres un terminal i et connectes al teu servidor per IP:

```bash
ssh joan@192.168.1.50
```

Se't preguntarà si confies en la «petjada digital» (fingerprint) del servidor. Això prevé atacs _Man-in-the-Middle_. Una vegada acceptat i proporcionada la teva contrasenya, et trobaràs davant la consola remota. Des d'aquest moment, pots desendollar físicament el teclat i el monitor del teu flamant servidor Debian.
