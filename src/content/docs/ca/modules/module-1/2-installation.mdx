Tot i que distribucions com Ubuntu ofereixen instal·ladors gràfics o semi-gràfics (TUI) molt senzills, un vertader SysAdmin ha de saber què ocorre sota el capó. Entendre una instal·lació manual (similar a com s'instal·la Arch Linux, però aplicada a conceptes de Debian) fonamenta el teu coneixement sobre com respira el sistema operatiu.

Quan instal·les un sistema Linux sense interfície gràfica, has d'ocupar-te de tres pilars fonamentals manualment: **el disc**, **els fitxers base** i **la xarxa**.

---

## 1. L'Esquema de Particionament

Abans de poder guardar un sol fitxer, necessites preparar el disc dur físic. En Linux, no existeixen les "unitats C: o D:". Existeix un únic arbre jeràrquic que comença a l'arrel (`/`), i els discs físics es "munten" (enganxen) en branques d'aquest arbre.

Per poder arrencar un sistema modern (UEFI), has de realitzar el següent esquema de particions mínim i professional:

<Steps>

1. **Crear la partició EFI (`/boot/efi`)**: Configura un espai petit (aproximadament 500MB) formatat en `FAT32`. És on el firmware de la placa base busca el gestor d'arrencada (GRUB) per encendre l'equip.

2. **Crear la partició Swap**: Assigna espai d'intercanvi usat com a "RAM d'emergència". Si la màquina es queda sense memòria física, usa aquest espai en disc. En servidors de bases de dades de vegades s'omet o es maneja com un fitxer, però tradicionalment se li assigna la seva pròpia partició (per exemple 2GB o la mida de la RAM).

3. **Crear la partició Arrel (`/`)**: Crea aquesta partició i formateja-la clàssicament en `ext4` o `xfs`. Contindrà tot el sistema operatiu.

</Steps>

En instal·lacions de grau empresarial, és comú separar `/home` (dades d'usuaris) o `/var` (bases de dades i logs) en particions o discs independents per evitar que un log descontrolat ompli el disc i impedeixi arrencar el sistema.

<Aside type="tip">
  **Eines CLI per Discs:** Durant una instal·lació manual, utilitzaràs
 comandaments com `fdisk /dev/sda` o `parted` per crear aquestes particions, i
  `mkfs.ext4 /dev/sda3` per donar-los format.
</Aside>

## 2. L'Arbre de Directoris (FHS)

Un cop el disc està formatat i muntat a `/`, es bolca (o descarrega) el sistema operatiu base. Linux segueix un estàndard anomenat _Filesystem Hierarchy Standard_ (FHS). Així és com s'organitza el teu disc:

<Steps>

4. **Familiaritzat amb els directoris del sistema**:
  - `/bin` i `/sbin`: Contenen els binaris (comandaments essencials com `ls`, `cp`, `ip`).
  - `/boot`: El Kernel (`vmlinuz`) i els fitxers d'arrencada inicial.
  - `/dev`: "Device". Fitxers especials que representen el teu maquinari. `/dev/sda` és el teu disc dur, `/dev/null` és el forat negre del sistema.
  - `/etc`: El cor de la configuració. **Si necessites canviar com es comporta un programa globalment, el fitxer de text està aquí.** (ex. `/etc/ssh/sshd_config`, `/etc/passwd`).
  - `/home`: Les carpetes personals de cada usuari regular (ex. `/home/juan`).
  - `/root`: La carpeta personal del superusuari (administrador suprem). Separada de `/home` per seguretat.
  - `/tmp`: Fitxers temporals. **S'esborra automàticament** en cada reinici. No guardis res important aquí.
  - `/usr`: Programes, biblioteques i utilitats no essencials per a l'arrencada instal·lats per l'usuari/sistema (`/usr/bin`).
  - `/var`: "Variable". Dades que creixen i canvien freqüentment. Aquí viuen els registres del sistema (`/var/log`), bases de dades i webs (`/var/www`).

</Steps>

<FileTree>
- / *Directori arrel del sistema*
  - bin/ Binaris essencials (`ls`, `cp`, `ip`...)
  - sbin/ Binaris del sistema per administració
  - boot/ Kernel (`vmlinuz`) i fitxers d'arrencada
    - efi/ Partició EFI — aquí viu el GRUB
  - dev/ Fitxers especials que representen maquinari (`sda`, `null`...)
  - etc/ **Cor de la configuració global del sistema**
    - ssh/
      - sshd_config
    - passwd
  - home/ Carpetes personals dels usuaris regulars
    - juan/
  - root/ Carpeta personal del superusuari (separada per seguretat)
  - tmp/ Temporals — **s'esborra en cada reinici**
  - usr/ Programes e utilitats no essencials per l'arrencada
    - bin/
    - lib/
  - var/ Dades variables que creixen amb el temps
    - log/ Registres del sistema
    - www/ Webs i bases de dades
</FileTree>

## 3. Configuració de Red per Consola (Wi-Fi i Ethernet)

Durant una instal·lació TUI (Text User Interface) o consola pura, lo més fàcil és connectar un cable Ethernet. El sistema usualment solicita una IP automàticament mitjançant DHCP (`dhclient eth0`).

¿Pero què passa si estàs en un servidor remot caser o un portàtil sense port Ethernet? **Necessites configurar el Wi-Fi des de la terminal de comandaments.**

### Usant NetworkManager (`nmcli`)

En la família Debian moderna, i especialment si eines comunitàries estan presents, la forma més estàndard per al LFCS de gestionar xarxes és mitjançant `nmcli` (Network Manager Command Line Interface).

<Steps>

5. **Revisar si la targeta Wi-Fi és detectada**:

   Mostra tots els dispositius de xarxa. Hauries de veure alguna cosa com `wlan0` o `wlp3s0` amb el tipus `wifi`.

   ```bash
   nmcli device status
   ```

6. **Escanejar xarxes disponibles**:

   Mostra les xarxes Wi-Fi a l'abast i la seva força de senyal.

   ```bash
   nmcli device wifi list
   ```

7. **Connectar-se a una xarxa**:

   Connecta i guarda la contrasenya per a futurs reinicis.

   ```bash
   nmcli device wifi connect "NomDeLaTevaXarxa" password "LaTevaContrasenyaSecreta"
   ```

8. **Verificar la IP assignada**:

   ```bash
   ip a
   ```

</Steps>

### Alternativa de baix nivell: `wpa_supplicant`

Si el medi d'instal·lació és extremadament mínim (com l'instal·lador clàssic de Debian o Arch), `NetworkManager` podria no estar instal·lat encara. En lloc d'això, s'utilitzen eines crues. Per això, es genera un fitxer de configuració injectant la teva contrasenya xifrada i després es crida al dimoni:

<Steps>

5. **Generar la configuració de la xarxa**:

   ```bash
   wpa_passphrase "NomDeLaTevaXarxa" "Contrasenya" > /etc/wpa_supplicant/wpa_supplicant.conf
   ```

6. **Iniciar la connexió en segon pla**:

   Assumint que la teva interfície és `wlan0`.

   ```bash
   wpa_supplicant -B -i wlan0 -c /etc/wpa_supplicant/wpa_supplicant.conf
   ```

7. **Sol·licitar una adreça IP**:

   ```bash
   dhclient wlan0
   ```

</Steps>

Amb internet funcionant, el sistema ja pot descarregar la resta de paquets (`apt install ...`) per finalitzar la instal·lació i reiniciar cap al teu nou Linux.
