## 1. Grups d'Usuaris

Tot usuari a Linux pertany, com a mûÙnim, a un grup.

### Grup Primari

### Grups Secundaris (Suplementaris)

Un usuari pot pertû nyer a cinquanta grups addicionals simultû niament (ex: grup `docker`, grup `www-data`, grup `sudo`).

### Operacions amb Grupos

- **Crear un grup nou**:
  ```bash
  sudo groupadd developers
  ```
- **Afegir un usuari existent a un grup suplementari**:
  ```bash

  # Compte! Si oblides la 'a', esborrarû s en Joan de tots els seus altres grups.
  sudo usermod -aG developers joan
  ```

- **Veure a quins grups pertany algû¤**:

## 3. Dominant el terror de `/etc/sudoers`

Com sap Linux exactament qui tûˋ dret a usar `sudo`? Perquû´ estû  escrit al sagrat fitxer `/etc/sudoers`.

Per aquest motiu crûÙtic, estû  TERMINANTMENT PROHIBIT editar `/etc/sudoers` amb Nano o Vim normals.
**SEMPRE has d'usar l'eina `visudo`**.

```bash

```

`visudo` obrirû  el fitxer en un editor. Perûý la mû gia passa en desar: `visudo` analitzarû  la sintaxi del teu text. Si has escrit alguna cosa incompatible que hagi de trencar el sistema, `visudo` bloquejarû  el desament i t'enviarû  un error, salvant la vida del servidor.

### Sintaxi de visudo

En baixar pel document, veurû s aquesta lûÙnia que li dû°na el poder de Dûˋu a l'usuari root universalment:
`root    ALL=(ALL:ALL) ALL`

Per donar accûˋs `sudo` a en Joan a tot el sistema:
`joan    ALL=(ALL:ALL) ALL`

Per donar accûˋs `sudo` a en Joan a tot el sistema, perûý **sense que se li demani contrasenya en executar comandaments** (molt û¤til per a scripts automatitzats):
`joan    ALL=(ALL:ALL) NOPASSWD: ALL`

`%developers    ALL=(ALL) NOPASSWD: /usr/sbin/service apache2 restart`
