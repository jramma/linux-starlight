Un servidor de producció aïllat i sense connexió a Internet és només una costosa caixa de metall escalfant l'habitació. Com a administrador LFCS, saber llegir o reparar la configuració de les targetes de xarxa físiques i lògiques de Debian és una obligació ineludible.

**Antiguitat:** Durant 20 anys usem eines com `ifconfig`, `route` o `netstat`. Actualment el projecte Linux els ha marcat com **obsolets** a favor de l'eina navalla-suïssa moderna: El comandament multiusos**`ip`**.

---

## 1. Veient les Interfícies i Ports (`ip a`)

El teu servidor pot tindre diverses targetes de xarxa físiques reals, i algunes virtuals. Cada una és una "Interfície".

Per saber quines IPs té configurades la teua màquina ara mateix, executa:

```
# Equivalent modern per a 'ifconfig' (ip address)
ip a
```

**El que veuràs**:

1. Una interfície anomenada**`lo` (Loopback)**: El sistema parlant-se habitualment a si mateix en `127.0.0.1`. Mai la toques.
2. Una interfície física anomenada**`eth0`, `enp0s3` o `ens33`**: La teua targeta de xarxa cablejada real. Sota la seua secció `inet` trobaràs la teua Adreça IP IPv4 d'àrea local (Ex. `192.168.1.150/24`).

## 2. La Porta d'Eixida: Enrutament (`ip r`)

A vegades el teu servidor té assignada una IP perfectament, però "no té Internet". Això ocorre perquè Linux no sap a quin router entregar-li les dades perquè isquen de l'edifici. A l'ordre que marca per on eixir al món se li diu "Ruta per defecte"o**Default Gateway**.

```
# Equivalent modern per a 'route' (ip route)
ip r
```

La línia més vital que has de buscar dirà alguna cosa com: `default via 192.168.1.1 dev eth0`. Això significa que si Linux necessita internet de fora, li mandarà els paquets al router (porta d'enllaç) `192.168.1.1`.

## 3. Diagnòstic Temprà (Pinging)

Si una web falla, el nostre protocol sempre ha de ser diagnosticar des de la capa més bàsica. ¿Acas el cable de xarxa està desconnectat?

El comandament `ping` usa el protocol ICMP de baix nivell. Bàsicament, tu crides_"¡Ecos!"_, i l'altra màquina (si està viva) et respon_"¡Estic ací!"_.

```
# Provar connexió contra els DNS públics de Google
ping 8.8.8.8

# Provar si el sistema de noms (DNS) aconsegueix convertir text a IPs reals
ping google.com
```

<Aside type="tip"title="Ping Infinit">
  A diferència de Windows, on el comandament `ping` es deté ell sol als 4
  intents, en Unix el `ping` no pararà MAI fins que tu l'assessines amb la
 combinació de cancel·lació forçada en terminal: **`Ctrl + C`**.
</Aside>

## 4. Estadístiques i Ports Escoltant (`ss`)

Imagina que el teu cap et crida paniquejant:_"¡Has instal·lat el servidor Apache però la web no carrega!"_. Tu vas fer el ping al servidor i sí respon. Llavors, ¿Acas el programa web Apache està corrent realment encés? ¿Està el port 80 (HTTP) obert i il·luminat per rebre al món?

Antigament usàvem el comandament `netstat`. Hui, els reis usen**`ss`**(Socket Statistics), que és mil vegades més ràpid.

```
# Llista processos escoltant la xarxa, resolent números en ports (ss tunel plain listen)
sudo ss -tulpn
```

Els paràmetres (TULPN) són el mantra del Sysadmin:

- `-t`: Mostrar tràfic**T**CP.
- `-u`: Mostrar tràfic**U**DP.
- `-l`: Solament components en estat `LISTEN` (Escoltant a què algú es connecte).
- `-p`: Mostrar el nom del**P**rocés culpable (`apache2` o `nginx`).
- `-n`: Mostrar els**N**ombres reals (Ex. port `80` en lloc d'imprimir un text que diga `http`).

Si corres este comandament i**NO**veus al teu programa llistat en la columna (Local Address:Port) com `0.0.0.0:80`, significa que el servidor web ni tan sols està arrancat en el sistema operatiu.
