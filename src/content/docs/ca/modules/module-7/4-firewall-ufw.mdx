## 1. L'Horror d'Iptables vs la Pau d'UFW

Com vam veure a la lliçó anterior (`ss -tulpn`), molts dimonis tendeixen per inèrcia o defecte a escoltar connexions dirigides cap a internet amb el perillós "0.0.0.0".  
Per evitar que un hacker connecti a aquests ports si el servidor està en producció, Linux posseeix un dels subsistemes de Kernel de tallafocs de xarxa més sofisticats i militaritzats de la terra: **`Netfilter / Iptables`**.

El problema radical corporatiu: la seva sintaxi és impossible de llegir i mantenir per a un humà amb pressa. (Tractar de tancar el port web implica comandaments infernals estil `iptables -A INPUT -p tcp --dport 80 -j DROP`).

Si t'equivoques en una lletra d' `iptables`, derroques la connexió de 1000 usuaris de la teva empresa en dos segons. Per simplificar aquesta monstruositat universal, Canonical (els amos d'Ubuntu) van crear un senzill "traductor" d'anglès amable anomenat **`ufw`**, el qual s'ha instal·lat com la norma d'or LFCS general.

## 2. El Ritual d'Activació UFW (¡Perill de mort!)

UFW ve apagat de forma nativa. La seva primera i principal configuració teòrica és la de tot guarda-barreres: **"Denegar TOT el tràfic entrant al PC, i Permetre TOT el tràfic sortint des del PC cap a internet"**.

<Aside type="caution" title="La Sentència de Mort del Teleballador">
Aprendràs l'error més llegendari (i còmic) que cometen el 100% dels Sysadmins Junior i Intermedis. 
Estàs a casa teletreballant en pijama connectant-te màgicament per SSH. Encens el tallafocs mitjançant `ufw enable`, el qual per defecte talla totes les connexions entrants (com dèiem abans).

¿Resultat? T'acabes de tallar el teu propi cordó umbilical i la teva pròpia connexió SSH cau per sempre al ser la teva sessió externa rebutjada instantàniament per UFW. Ja no pots accedir a treballar avui sense anar en persona al centre de dades (patint humiliació corporativa).
**La regla d'or: Abans d'encendre la muralla, assegura't de perforar un forat per a la teva pròpia corda SSH sempre.**

</Aside>

### 1. Perforant la xarxa abans del desastre

```bash
# Instal·lem si no existeix al servidor mínim
sudo apt install ufw

# Fem el súper túnel blindat per no tallar-nos el dit
sudo ufw allow ssh
```

### 2. Encesa Activa del Tallafocs

```bash
# Encens la muralla (es recordarà després de cada reinici també)
sudo ufw enable

# Veiem el nostre estat actual per confirmar quins túnels existeixen
sudo ufw status verbose
```

## 3. Esculpint les Defenses de Xarxa (Regles)

Ara que la nostra muralla UFW rebotarà tota la brossa externa d'internet, ens toca permetre perforacions puntuals o forats (`allow`) perquè la gent de negocis pugui treballar en els ports dels Dimonis interns.

**Permetre accés web general (`allow`)**
Podem parlar-li mitjançant noms de serveis coneguts o mitjançant número de port i el protocol:

```bash
sudo ufw allow 80/tcp
# Equivalent a:
sudo ufw allow http
```

**Restringir orígens elitistes a IPs secretes**
Suposa que instal·les una Base de Dades (`3306`). Mai de la vida sota cap circumstància la obris al públic universal amb un _allow 3306_ normal. Per a això, crees un forat làser que _només s'obri_ si la trucada a la porta procedeix de l'adreça IP del teu servidor web a la IP `10.0.0.5`:

```bash
# Només admetem el nostre amic al port 3306
sudo ufw allow from 10.0.0.5 to any port 3306
```

**Bloquejos Explícits Selectius (`deny` o `reject`)**
Hi ha un troll atacador al carrer a la màquina concreta `198.51.100.1` que et bombardeja la web i vols fer-lo fora independentment de la regla permesa:

```bash
sudo ufw deny from 198.51.100.1
```

## 4. La Trampa Anti-Força Bruta: `limit`

`ufw` porta incorporat un mòdul de defensa automàtica brutal per a dimonis ultra-sensibles (com SSH al `22`).
Si usem la regla especial de `limit` en comptes d' `allow` o `deny`, UFW construirà matemàticament per darrere al Kernel una bossa contenidora. Si algú (tu o un bot xinès) intenta loguejar-se per SSH **6 o més vegades fallides en una diminuta finestra de 30 segons**, UFW ho catalogarà silenciosament com a bot de diccionari i li cremarà la IP negant-li futures connexions en els propers dies.

```bash
# El comandament defensiu suprem de l'administració professional remota
sudo ufw limit ssh
```
