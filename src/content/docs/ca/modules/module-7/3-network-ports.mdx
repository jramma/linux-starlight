## 1. L'Anatomia dels Ports

Imagina que el teu servidor és un edifici corporatiu enorme amb l'Adreça IP `192.168.1.10`.
Si un ordinador extern intenta enviar un paquet a aquest edifici, el paquet arriba a la porta principal i xoca contra un mur. ¿A qui anava dirigit el paquet? ¿Al servidor web? ¿Al servidor de correu? ¿Al xat?

Per solucionar això existeixen **Els Ports** (Numerats típicament de l' `1` al `65535`). Són com les "extensions telefòniques" de l'edifici.

- Tot paquet web arriba a l'extensió **`80` (HTTP)** o **`443` (HTTPS)**.
- Tot protocol de terminal remota arriba al **`22` (SSH)**.
- Tot paquet de correu arriba al **`25` (SMTP)**.

Els dimonis (serveis com systemd dels quals hem parlat tot el curs), quan s'encenen, s' **"Atan" (Bind)** a un port, quedant-se de braços plegats, congelats _"en modo escolta"_ (`LISTEN`), esperant a que algú els truqui.

### La Perillosa Pregunta LFCS

_¿Com sé jo quins dimonis estan ara mateix escoltant portes a les meves esquenes i cap a quines xarxes estan oberts?_

## 2. El Radar Modern (`ss`)

En l'antiguitat s'usava el comandament `netstat`. Avui dia (i al teu examen LFCS), està obsolet per lent i ha estat reemplaçat per **Socket Statistics (`ss`)** del paquet de xarxes que vam veure al mòdul 3 (`iproute2`).

`ss` a pèl et llançaria un milió de línies amb cada connexió interna de xarxa. Els Sysadmins usem sempre la mateixa composició de lletres inamovibles: **`-tulpn`**

```bash
sudo ss -tulpn
```

**¿Què signifiquen les màgiques `-tulpn`?**

- **-t**: Mostra connexions TCP (Web, SSH, bases de dades).
- **-u**: Mostra connexions UDP (Coses ràpides sense confirmació d'arribada com DNS o Videojocs).
- **-l**: _¡VITAL!_ Només mostra dimonis lligats en modo `LISTEN` i oculta tot el soroll de connexions ja establertes d'usuaris.
- **-p**: _¡VITAL!_ Mostra el nom humà del Programa PID que és amo d'aquest port. Si no poses la 'p', només veuràs números. (Requereix Sudo).
- **-n**: Numeric. Evita que Linux intenti traduir "22" a la paraula "ssh", mostrant-nos els números crus dels ports reals.

### Comprenent la sortida de `ss`

Suposem que veus això:
`tcp   LISTEN 0      128    0.0.0.0:80        0.0.0.0:*      users:(("nginx",pid=1055,fd=6))`

Significa que **Nginx** (pid 1055) té l'amo del port web `80` per TCP.
El més important és la part `0.0.0.0`. Aquesta és l' "Adreça lligada".

- `0.0.0.0` significa: **"Escolto peticions venint des de qualsevol part de l'univers i d'Internet"**. (Perillós).
- Si posés `127.0.0.1:80` (Localhost) significaria: **"Només escolto peticions segures d'altres programes instal·lats dins d'aquest mateix ordinador, sóc cec cap a internet"**.

## 3. Investigador Privat (`lsof`)

Si `ss` és la visió general del país, _List Open Files (`lsof`)_ és el làser mil·limètric enfocat a una sola ciutat.

<Aside type="tip" title="L'estàndard Unix">
  Recorda que a Linux *tot és un fitxer*. Si Chrome obre una connexió xarxa a
  internet cap a un vídeo de youtube, per al Kernel, aquesta connexió viva de xarxa
  és un fitxer obert més com si fos un .txt
</Aside>

Si instal·les el servidor Apache, i intentes engegar-lo amb `systemctl start apache2`, a vegades rebenta llançant l'error `Port 80 is already in use` (El port 80 ja està en ús).
¿Quin altre dimoni et va robar el port a les teves esquenes abans d'engegar tu l'Apache? `ss` podria mostrar-t'ho, però `lsof` és explícitament diví per a això:

```bash
# ¿Qui dimonis (-i internet) està agafant el meu port de dos punts (:80)?
sudo lsof -i :80
```

Aquest simple comandament bolcarà el culpable (per exemple, potser vas deixar un Nginx vell encès per error) llistant l'usuari exacte (`root`, `www-data...`) perquè vagis i aturis el conflicte matant el seu PID amb `kill -9 1055` o aturant el servei amb `systemctl stop nginx`.
