## 1. El Fracàs dels Permisos Clàssics (DAC)

Al Mòdul 2 d'aquest curs, vam invertir hores aprenent com usar `chmod` i `chown` per establir permisos "rwx" (Lectura, Escriptura, Execució) en fitxers. Tot aquest sistema arcaic mundialment utilitzat s'anomena **DAC (Discretionary Access Control o Control Discrecional)**.

S'anomena "Discrecional" perquè la seguretat queda a **discreció dels usuaris**.
_Imagina que ets un caixer de banc normal. Tens permís de Lectura per veure una caixa forta secreta. Usant els sistemes DAC habituals, com tu "posees" el fitxer, podries decidir discretament donar permís a tot l'univers (`chmod 777`) perquè qualsevol persona a internet pogués accedir a la caixa forta del banc._

### El Risc amb els Dimonis (Serveis Hackejats)

Si executes un servidor web com Nginx sota l'usuari arrel de sistema `root` i un hacker rus a internet esbrina un error en la programació de Nginx per trencar la pàgina web i injectar el seu propi codi... Nginx executarà el codi del Rus de bona fe amb el poder infinit dels permisos `root`, controlant absolutament tota la CPU i destruint-te.

Per frenar els programes desbocats o els propis usuaris que es donen permisos perillosos, en alta seguretat corporativa usem **MAC**.

## 2. El Dictador de Kernel: MAC i AppArmor

**MAC (Mandatory Access Control o Control Obligatori)** rebutja l'opinió dels usuaris. Aquí defineix les regles un dictador inqüestionable des del més profund del propi Kernel Linux.

A Debian (i Ubuntu), aquest dictador s'anomena **AppArmor** (A RedHat s'anomena SELinux).

Sota MAC/AppArmor, als programes se'ls posa una "Camisa de Força" especial matemàtica en forma de fitxer de text anomenada **Perfil**.
Si el kernel de Debian té carregat un "Perfil d'Apparmor" especial per a Nginx que diu _"Nginx MAI sota cap circumstància pot usar eines del grup Bash"_, és igual que configuris el programa amb permisos Root, és igual que fiquis Nginx a usar eines d'administració, i és igual que el hackegin: en el moment en què Nginx intenti executar alguna cosa prohibida al Perfil, AppArmor invocarà el martell de Thor bloquejant l'acció a nivell Zero i enviant-te una alerta silenciosa.

## 3. Comandaments vitals com a Auditor LFCS

AppArmor ja ve instal·lat en sistemes comercials en l'actualitat i porta "Perfils prefabricats" de camisa de força llistos i actius en RAM per encapsular al 90% del programari crític instal·lable de Debian.

Els binaris d'auditoria comencen per `aa-` (App Armor).

### Veure els tancaments actius

La primera tasca d'un sysadmin al matí:

```bash
sudo aa-status
```

Això llançarà dues llistes molt diferenciades:

1. **Perfils carregats en modo `Enforce`**: La seva camisa de força obliga físicament. Si un programa d'aquesta llista trepitja el prohibit, serà aniquilat instantàniament.
2. **Perfils carregats en modo `Complain`**: Estan sent vigilats. Si creuen la línia, el perfil t'apuntarà silenciosament "Ei, aquest programa acaba de fer coses rares a la teva carpeta home", però **NO aturarà la seva execució**. (Generalment usat per programadors per diagnosticar els seus propis programes durant 1 setmana abans de posar la seva camisa local de fàbrica en Enforce).

### Administrar els Modos dels Fitxers

Si vols emmudir les regles letals d'un programa de bases de dades `mysqld` perquè no para de tirar-se i creus que un fals positiu ha trencat alguna cosa després d'una actualització, pots empènyer-lo temporalment i calmadament a modo "renocs" (Complain).

```bash
# Empenyent al modo Xevat/Renoc pacífic
sudo aa-complain /usr/sbin/mysqld

# Tornant a restablir la dictadura marcial castrense
sudo aa-enforce /usr/sbin/mysqld
```

\*_(NOTA: Els perfils complets, escrits amb sintaxi arcana, solen allotjar-se tots junts estàtics a la caixa forta local a `/etc/apparmor.d/`)_.\*
