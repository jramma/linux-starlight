## 1. El Perill del Temps

Per a un ordinador personal, tenir el rellotge endarrerit 5 minuts només significa que arribaràs tard a una reunió. **Per a un Servidor de base de dades, 5 minuts d'endarreriment signifiquen la destrucció del negoci:**

- **Criptografia Trencada**: Si un servidor teu intenta connectar-se a la passarel·la de pagaments del banc, i el banc detecta que el rellotge del teu servidor diu que són les 15:00 mentre que al banc són les 15:05, el banc catalogarà la connexió com un "Atac de Replay" i tallarà les teves transaccions financeres en sec.
- **Logs Inservibles**: Si pateixes un hackeig, i un servidor registra l'atac a les 11:00 i l'altre servidor el registra a les 11:02 perquè no estaven sincronitzats entre si, és impossible reconstruir l'autòpsia de per on va entrar el hacker.

## 2. Els Dos Rellotges

Tot ordinador té en realitat **dos rellotges diferenciats** que barallen entre sí:

1. **Rellotge de Maquinari (Real Time Clock - RTC)**: El xip físic alimentat per la pila de botó de la placa base.
2. **Rellotge del Sistema (Programari)**: El rellotge imaginari que Linux comença a comptar a la memòria RAM només engegar-se llegint inicialment la BIOS.

Podem obligar-los a parlar-se:

```bash
# Veure l'hora real de la diminuta BIOS de la placa base
sudo hwclock --show

# Obligar a la BIOS a ajustar-se forçosament a l'hora que tingui el Linux ara mateix
sudo hwclock --systohc
```

## 3. Configurar la Franja Horària (Timezone)

Per defecte, els servidors moderns corporatius sempre es configuren en **UTC (Universal Time Coordinated)**, sense retards ni avançaments ni canvis d'estiu, perquè tots els servidors del món parlin el mateix idioma en els Logs.

Així i tot, si necessites forçar l'hora local de Madrid o Mèxic, l'eina obligatòria de Systemd és `timedatectl`:

```bash
# Veure el resum i estat de l'hora actual
timedatectl status

# Cercar com s'escriu el nom oficial de Madrid
timedatectl list-timezones | grep Madrid

# Assignar la zona horària perpètuament (Reemplaça l'antic /etc/localtime)
sudo timedatectl set-timezone Europe/Madrid
```

## 4. NTP: El Batec Global (`chrony`)

En lloc d'ajustar l'hora a mà com en un rellotge de polsera, els servidors utilitzen el màgic **Network Time Protocol (NTP)**. NTP viatja per internet, es connecta a rellotges atòmics controlats per universitats i governs, i calcula el retard que genera el cable submarí d'internet per ajustar l'hora de la teva màquina a l'absolut mil·lisegon perfecte de manera constant i paramètrica cada pocs minuts.

En el vell estàndard s'usava un paquet anomenat `ntp`. Al món LFCS modern corporatiu d'avui s'exigeix usar la seva versió ràpida i superior: **`chrony`**.

```bash
# Instal·lem el motor de rellotge atòmic
sudo apt install chrony

# Iniciar i assegurar que corre cada vegada que el servidor s'engegui
sudo systemctl enable --now chronyd
```

Si vols veure com de bé s'està comunicant el teu servidor amb el món exterior:

```bash
# Monitoritza els servidors atòmics mundials als quals estàs enllaçat
chronyc sources -v
```

_(Si veus línies amb símbols de `^_`, significa que estàs màgicament enganxat i el teu rellotge MAI s'endarrerirà ni un segon)._
