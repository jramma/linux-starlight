## 1. Benvinguts a l'Infern (Filosofia LFCS)

A la certificació LFCS (i al món empresarial real), gairebé **mai et demanen instal·lar coses noves**. Els servidors ja estan instal·lats per una altra persona que es va acomiadar fa tres mesos.
El teu examen consistirà a accedir per consola a 15 servidors diferents, tots amb un problema catastròfic subjacent diferent que provoca que l'aplicació no funcioni, i et donaran 2 hores de rellotge de sorra per arreglar-los tots.

No hi ha internet per buscar a Google. No hi ha apunts. Estàs tu davant del Kernel.

Aquí tens tres dels escenaris més temuts i clàssics.

## 2. Escenari 1: El Servidor de Base de Dades està Tombat

Rells una alerta a les 4:00 AM. La pàgina web llança un terrorífic _"Error Establishing a Database Connection"_. Entres per SSH al servidor de base de dades i comprovem que el dimoni MariaDB ha mort. Intentes arrencar-lo tirant `sudo systemctl start mariadb` i falla.

**¿Què faries?**

<Aside type="caution" title="L'instint de supervivència LFCS">
  Un novell intentaria desinstal·lar i reinstal·lar la base de dades `apt remove
  mariadb` destruint l'empresa sencera. El LFCS acudeix al **Diari d'Abat
  (Mòdul 4)** purificant per prioritat d'error crític: `journalctl -u
  mariadb -p err -n 10`
</Aside>

En aquell precís instant, el log et revela l'assassí real:
_"Fatal error: Can't create/write to file '/var/log/mysql/... (Errcode 28: No space left on device)"_.

¡La base de dades s'ha apagat per protegir-se perquè el disc dur està físicament ple al 100%!

### La Solució LFCS (LVM al rescat - Mòdul 5)

Com que som previsors, el servidor es va forjar usant la virtualització d'emmagatzematge LVM.

1. Executem `vgs` (Volume Group Status) i comprovem que a la "piscina" màgica global sobren 50 Gigabytes sense usar.
2. Injectem 10 Gigues al disc virtual root en calent forçant el redimensionament del file system ext4 simultàniament:
   ```bash
   # Mòdul 5 pur: lvextend injecta el plàstic, -r el motlla.
   lvextend -L +10G -r /dev/mapper/debian-vg-root
   ```
3. Tornem a aixecar el servei `sudo systemctl start mariadb`. Sanat espectacularment en 20 segons sense reiniciar.

## 3. Escenari 2: El Segrest Cec del Port web

Arribes a una empresa nova el teu primer dia de feina. Et diuen que reiniciïs Nginx al servidor central. L'apagues amb `systemctl stop nginx` i quan vas a encendre'l amb `systemctl start nginx` t'escup la infame fallada `[emerg] bind() to 0.0.0.0:80 failed (98: Address already in use)`.

Algú (o alguna cosa) ha encès un altre programa a les teves esquenes en els últims 5 segons que t'ha robat l'extensió telefònica del port 80 impedint que Nginx pugui existir. Et suen les mans perquè la pàgina web principal de l'empresa està caiguda.

### La Solució LFCS (SS i lsof - Mòdul 7)

Tu no et poses a buscar fitxers lògics. Tu traus els raigs X del Mòdul 7.

1. Uses l'escàner làser inamovible buscant la mosca directament en aquell socket d'internet específic:
   ```bash
   sudo lsof -i :80
   ```
2. El comandament devela brutalment que el programa invasor és el vell contenidor `apache2` i que se li ha assignat el PID diabòlic `4591`.
3. El erradiques sense pietat del mapa de processadors amb el senyal 9 d'assassinat indiscutible (Mòdul 2).
   ```bash
   sudo kill -9 4591
   # Alternativa civilitzada: sudo systemctl stop apache2
   ```
4. Crees lliure i ressuscites la màquina web amb `sudo systemctl start nginx`.

## 4. Escenari 3: "He oblidat la meva contrasenya" (Atac físic)

Escenari final de l'examen LFCS. Et tiren a un terminal negre d'un servidor i ni tan sols et donen la clau d'inici de sessió de ROOT ni del teu usuari. Volen que els demostris que saps com bypassejar les seves defenses si tens accés al metall físic de la màquina.

### La Solució LFCS (L'Hackeig de GRUB - Mòdul 4)

1. Força un reseteig de la màquina virtual apagant-la a matxet.
2. En els dos únics segons que dura la pantalla lluminosa del menú del carregador d'arrencada GRUB, prem frenètic la fletxa cap avall aturant el compte enrere i posa't sobre la paraula **"Debian GNU/Linux"**.
3. Pressiona la terrorífica tecla injectora `e` (Edit).
4. Busca amb les fletxetes la densa línia kilomètrica que comença amb la paraula clau `linux`. Baixa fins al pur final d'aquella mateixa línia de merda, i col·loca-li separant-ho temporalment el bypass del Kernel pasmós:
   `init=/bin/bash`
5. Prem `Ctrl + X` (o `F10`) per forzar una arrencada màgica desoint els segells divins. El PC encendrà atorgant-te un prompt fred `#` cedint-te el tron Root pur però amb el disc dur congelat en modo "Només lectura".
6. Descongela el gel muntant per sobre el disc local amb capacitats re-escriptores de modificació i canvia-la:
   ```bash
   mount -o remount,rw /
   passwd root
   ```
7. Pous la nova clau "1234", dónes l'ordre de recongelament pacífic (`exec /sbin/init`) i aproves el rescat loguejant-te en pau amb la teva clau nova.

## Comprova els teus coneixements

1. Examen final: Un servidor arrenca. Tot sembla bé. Tires un `ping google.com` i rebenta retornant-te _"Temporary failure in name resolution"_" (Fallada de salt de noms DNS). Sabent que tens IP en rang gràcies a un cable Ethernet connectat, configures un Tallafocs UFW permetent sortida universal, i el Gateway (router) de l'empresa està responent les teves peticions IP internes però segueixes sense poder resoldre lletres de dominis. L'examen LFCS se'n riu de tu. Quin document nuclear primitiu has d'obrir o forçar a debian per injectar-li el servidor satel·litari global (Ex: Un `nameserver 1.1.1.1` o `8.8.8.8`) per parchejar aquest enverinament resolutiu?

   <MultipleChoice>
     <Option isCorrect>
       El pilar resolutiu local històric innegociable editant:
       `/etc/resolv.conf` (O re-configurant-lo indirectament a través del
       diable `systemd-resolved`). Si aquest fitxer conté IPs caducades, DNS
       locals morts o l'han buidat amb un `rm`, el cervell de Linux es
       tornarà boig i serà cec incapaç universalment de vincular paraules
       hermoses a números IP per molt excel·lent ruteig local que hi hagi.
     </Option>
     <Option>
       El pilar UFW inserint la regla DNS: `sudo ufw block resolv 53`
     </Option>
     <Option>
       Corrent el gestor de paquets de baix nivell `dpkg -i fix-dns.deb`.
     </Option>
   </MultipleChoice>
