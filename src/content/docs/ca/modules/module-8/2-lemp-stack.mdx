## 1. Deixant enrere l'edat de pedra

A la lliçó anterior vam encendre un Nginx capaç de mostrar un solitari fitxer `index.html`.
El problema el 2026 és que cap pàgina corporativa real usa HTML estàtic cru dictat a mà. Les empreses empren Gestors de Contingut monstruosos (com WordPress) o ERPs (com Odoo), els quals estan programats purament usant Llenguatges Dinàmics (com **PHP** o **Python**) i emmagatzemen milions d'usuaris en **Bases de Dades relacionals** estructurades.

A l'ecosistema Sysadmin mundial estàndard conformat per **L**inux, **E**ngine-X (Nginx), **M**ariaDB i **P**HP se'l coneix corporativament com la infraestructura **Stack LEMP**.

---

## 2. El Processador de Llenguatge (`PHP-FPM`)

Nginx és un geni servint fitxers passius a la xarxa (com fotos o HTML), però és un inútil intentant entendre codi de llenguatges de programació.
Si un usuari intenta visitar `http://empresa.com/login.php`, Nginx no té ni idea de processar-lo i, per defecte, es rendeix i ofereix a l'usuari la "descàrrega freda" de tot el codi de programació intern al seu disc local. ¡Un forat de seguretat brutal!

Per solucionar això existeixen els **FPM (FastCGI Process Managers)**. Bàsicament són dimonis de sistema (`systemctl`) que corren independentment amagats al fons escoltant ordres.

```bash
# Instal·lem la navalla suïssa d'execució ràpida asintrònica php-fpm
sudo apt install php-fpm
```

### Configurant Nginx perquè delegui peticions

A Nginx cal instruir-lo perquè derivi la feina difícil al nostre acabat d'instal·lar obrer de fons. Al fitxer `sites-available/empresa.com` afegirem un bloc "Location" de francotirador especial que diu: _"En l'instant en què vegis un fitxer amb terminació \.php, ni el miris, envia'l per un conducte TCP local al nostre amic FPM instal·lat ara, ell et retornarà l'HTML mastegat"_.

```nginx
server {
    listen 80;
    server_name mi-empresa.com;
    root /var/www/mi-empresa.com/html;
    # Diu que prefereixi corre index.php primer si existeix
    index index.php index.html;

    # ¡El conducte Socket mestre!
    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        # Llançant la pilota de feina al socket unix ultraràpid de l'obrer FPM versió 8
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
    }
}
```

## 3. El Cor Transaccional (`MariaDB`)

L'últim pilar de la Pila LEMP recau sobre l'emmagatzematge elàstic. Substituirem l'obsoleta deïtat original (MySQL) per la seva alternativa comunitària Open Source superior, estandarditzada actualment als servidors de tot el globus terraqüi: **MariaDB**.

```bash
# El baixarà i arrencarà el servei automàticament
sudo apt install mariadb-server

# L'script LFCS que TANCA portes vulnerables de fàbrica
sudo mariadb-secure-installation
```

_(Aquest meravellós script d'assegurament t'obligarà interactivament a col·locar-li clau a l'administrador mestre ("Root"), esborrarà bases de dades "Test" inútils i prohibirà brutalment que algú esbrini la teva IP pública i es connecti a la base de dades sense estar loguejat localment)._

## 4. Auditoria Post-Desplegament

La complexitat es multiplica en apilar serveis dispars (`Nginx`, `PHP`, i `MariaDB`), tots lligats fortament pels seus permisos del Mòdul 2 a l'usuari mestre estandarditzat de la web `www-data`.
Si després de realitzar l'assemblatge a l'examen LFCS de l'Stack el navegador vomita una fatal **Pàgina Blanca de la Mort (Error HTTP 500)**, ¡mai toquis botons ni t'inventis solucions resant!

1. Demana-li al dimoni Web que et narri els horrors que ha viscut:
   `journalctl -u nginx --no-pager | tail -n 20`
2. Ves al diari sanguini local que guarden els obrers i revisa'l amb els comandaments clàssics:
   `tail -f /var/log/nginx/error.log`

Trobàs respostes precises com _"php-fpm manca del driver mariadb-php per connectar al cor del SQL"_ o _"Permís denegat en obrir fitxer var/www/..."_.
