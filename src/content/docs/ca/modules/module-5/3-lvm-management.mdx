## 1. L'Ocap de les Particions Clàssiques

Imagina que crees un servidor usant el mètode de la lliçó anterior: Compres un disc dur de 50GB i el formates (`/dev/sda1`). Mesos després, la base de dades d'usuaris de la teva empresa creix a 49GB. El teu cap compra espantat un disc de 100GB i te'l dóna perquè l'endollis.

¿I ara què? La teva base de dades només sap escriure en una sola carpeta (`/var/lib/mysql`).
Com que ambdós discs són trossos de plàstic aïllats l'un de l'altre, **no pots unir l'espai dels dos**. Hauries d'aturar el servidor, copiar 49GB d'un disc a l'altre, reconfigurar la base de dades, repassar muntatges i resar per no corrompre res.

**Solució 100% vitalícia:** LVM (Logical Volume Management).
LVM és una capa de màgia per programari. LVM agafa tots els discs físics de plàstic que tu li fiquis, els fon en una sola súper-piscina gegant de megabytes, i et permet "retallar" discs imaginaris (Volums Lògics) d'aquesta piscina. ¿Se t'omple la base de dades? Li afegeixes 20GB extres imaginaris sobre la marxa sense aturar-la.

## 2. La Tríada Estricta de LVM

Sempre cal seguir tres passos matemàtics inqüestionables: Físic, Grup, i Lògic.

Suposem que acabes de connectar dos discs durs totalment verges: `/dev/sdb` (10GB) i `/dev/sdc` (20GB).

### Pas 1: Volums Físics (PV - Physical Volume)

Agafem els discs verges i els "marquem" amb el segell de LVM per indicar-los que formaran part d'un ecosistema virtual.

```bash
sudo pvcreate /dev/sdb /dev/sdc

# Comprovar la feina
sudo pvs
```

### Pas 2: Grup de Volums (VG - Volume Group)

Creem la "Piscina gegant". Crearem una piscina anomenada `magatzem_dades` i li llançarem els dos discs marcats.

```bash
sudo vgcreate magatzem_dades /dev/sdb /dev/sdc

# Comprovar. Si mires la mida, veuràs que la piscina ara té 30GB totals (10+20).
sudo vgs
```

### Pas 3: Volums Lògics (LV - Logical Volume)

Ara som déus. Usem una escumadora per treure 15GB d'aquesta piscina (no importa físicament en quin disc acabin) per crear la nostra partició final invisible que li dóna a la base de dades.

```bash
# Crear un volum lògic (-n) anomenat BD de mida (-L) 15GB des del grup magatzem_dades.
sudo lvcreate -n BD -L 15G magatzem_dades

# Comprovar
sudo lvs
```

¡Llest! Per a la fase de la lliçó anterior (`mkfs` i `mount`), a Linux ja no li importa `sdb`. Actuarà com si aquest xiclet virtual fos un disc sòlid anomenat: `/dev/magatzem_dades/BD`.

## 3. Màgia Negra Sysadmin: Lvextend

Tornem a l'exemple: La base de dades, muntada sobre l'LVM, arriba a 14.9 GB de 15 GB. El servidor està a punt d'esclatar d'errors.

Mirem amb `vgs` si ens sobra una mica de líquid màgic a la piscina `magatzem_dades`. Veiem que com que vam ficar 30GB originals i només en vam gastar 15GB, ens en sobren 15GB. Anem a injectar-li 5GB de rescat als discs de la base de dades, **mentre la base de dades insereix factures i sense muntar res**.

```bash
# Estenem la frontera del contenidor elàstic (+5G) al nostre Volum Lògic
sudo lvextend -L +5G /dev/magatzem_dades/BD

# Màgia profunda: LVM ha crescut en plàstic a 20GB. PERÒ el formatador ext4
# segueix creient cecament que la seva casa mesura 15GB. Obliguem al Format a recalcular:
sudo resize2fs /dev/magatzem_dades/BD
```

_(Si vas formatar amb XFS, el segon comandament màgic de forçat no és `resize2fs`, és `xfs_growfs`)._
