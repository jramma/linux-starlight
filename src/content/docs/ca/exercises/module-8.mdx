## Context — Què és la pila LEMP?

LEMP és l'acrònim de:
- **L**inux — el sistema operatiu (ja el tens)
- **E**ngine-X (nginx) — el servidor web que rep les peticions HTTP
- **M**ariaDB — la base de dades relacional
- **P**HP — el llenguatge que genera les pàgines dinàmiques

És una de les combinacions més usades en servidors web de producció. WordPress, Drupal i moltes aplicacions corren sobre LEMP.

## Pas 1 — Preparar el servidor

```bash
# Assegura't que el sistema està actualitzat
sudo apt update && sudo apt upgrade -y

# Crea els directoris de treball
sudo mkdir -p /var/www/misite
sudo mkdir -p /backup/db

# Verifica l'estat del tallafocs (ha d'estar actiu des de la pràctica 7)
sudo ufw status
```

## Pas 2 — Instal·lar nginx

```bash
sudo apt install -y nginx

# Inicia i habilita el servei
sudo systemctl start nginx
sudo systemctl enable nginx

# Verifica que respon
curl -s -o /dev/null -w "nginx respon: HTTP %{http_code}\n" http://localhost
# Ha de retornar: HTTP 200
```

Si el tallafocs està actiu, permet el trànsit web:

```bash
sudo ufw allow 'Nginx Full'
# Això obre els ports 80 (HTTP) i 443 (HTTPS)
```

## Pas 3 — Instal·lar MariaDB

```bash
sudo apt install -y mariadb-server

# Inicia i habilita
sudo systemctl start mariadb
sudo systemctl enable mariadb

# Assegurar la instal·lació: estableix contrasenya de root, elimina usuaris anònims
sudo mysql_secure_installation
```

Durant `mysql_secure_installation`:
- Enter current password for root: prem **Enter** (no hi ha contrasenya encara)
- Switch to unix_socket authentication: **n**
- Change the root password: **y** → tria una contrasenya segura i anota-la
- Remove anonymous users: **y**
- Disallow root login remotely: **y**
- Remove test database: **y**
- Reload privilege tables: **y**

Verifica que funciona:

```bash
sudo mysql -u root -p
# Introdueix la contrasenya que acabes de crear
# Ha d'aparèixer el prompt: MariaDB [(none)]>
exit
```

## Pas 4 — Crear la base de dades i l'usuari

```bash
sudo mysql -u root -p
```

Al prompt de MariaDB, executa aquests comandaments un a un:

```sql
-- Crea la base de dades
CREATE DATABASE cursodb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Crea l'usuari (posa la teva pròpia contrasenya)
CREATE USER 'cursouser'@'localhost' IDENTIFIED BY 'CursoLinux2024!';

-- Concedeix tots els permisos sobre la base de dades a l'usuari
GRANT ALL PRIVILEGES ON cursodb.* TO 'cursouser'@'localhost';

-- Aplica els canvis
FLUSH PRIVILEGES;

-- Verifica que la BD existeix
SHOW DATABASES;

-- Verifica que l'usuari existeix
SELECT user, host FROM mysql.user WHERE user = 'cursouser';

exit
```

Prova que l'usuari es pot connectar:

```bash
mysql -u cursouser -p cursodb
# Introdueix la contrasenya de l'usuari cursouser
# Ha d'aparèixer el prompt: MariaDB [cursodb]>
exit
```

## Pas 5 — Instal·lar PHP-FPM

```bash
sudo apt install -y php-fpm php-mysql php-cli

# Verificar la versió instal·lada i guardar-la (la necessitaràs després)
php --version
PHP_VERSION=$(php -r "echo PHP_MAJOR_VERSION.'.'.PHP_MINOR_VERSION;")
echo "Versió PHP: $PHP_VERSION"

# El servei s'anomena php8.X-fpm (X varia segons la versió)
sudo systemctl start php${PHP_VERSION}-fpm
sudo systemctl enable php${PHP_VERSION}-fpm

systemctl is-active php${PHP_VERSION}-fpm
# Ha de dir: active
```

## Pas 7 — Configurar el virtualhost de nginx

```bash
sudo nano /etc/nginx/sites-available/misite
```

Escriu (reemplaça `8.X` per la teva versió de PHP real, per exemple `8.3`):

```nginx
server {
    listen 80;
    listen [::]:80;

    server_name misite.local;
    root /var/www/misite;

    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
    }

    location ~ /\.ht {
        deny all;
    }
}
```

<Aside type="note" title="Com sé la meva versió de PHP?">
Executa `php --version` i mira el número. Si és PHP 8.3, usa `php8.3-fpm.sock`. Si és 8.2, usa `php8.2-fpm.sock`.
</Aside>

Activa el virtualhost i desactiva el de per defecte:

```bash
# Crea l'enllaç simbòlic per activar el lloc
sudo ln -s /etc/nginx/sites-available/misite /etc/nginx/sites-enabled/

# Desactiva el lloc per defecte (opcional però recomanat)
sudo rm /etc/nginx/sites-enabled/default

# Verifica que la configuració de nginx és correcta (sense errors de sintaxi)
sudo nginx -t
# Ha de dir: syntax is ok / test is successful

# Recarrega nginx per aplicar la nova configuració
sudo systemctl reload nginx
```

## Pas 10 — Backup automàtic de la base de dades

```bash
# Prova manual primer
sudo mysqldump -u root cursodb > /backup/db/cursodb_$(date '+%F').sql

# Verifica que el fitxer s'ha creat
ls -lh /backup/db/
```

Automatitza amb cron:

```bash
# Edita el crontab de l'usuari root
sudo crontab -e
```

Afegeix aquesta línia al final (s'executa a les 2:00 AM cada dia):

```
0 2 * * * mysqldump -u root cursodb > /backup/db/cursodb_$(date '+\%F').sql
```

<Aside type="note" title="Per què \%F i no %F en cron?">
A crontab, el caràcter `%` té significat especial i s'ha d'escapar com a `\%`.
</Aside>

Verifica que el cron està desat:

```bash
sudo crontab -l
```

## Pas 11 — Securitzar el servidor

Aplica les mesures de seguretat apreses:

```bash
# 1. Verifica UFW (de la pràctica 7)
sudo ufw status verbose

# 2. Verifica fail2ban
sudo fail2ban-client status sshd

# 3. Verifica que root no es pot connectar per SSH
grep "PermitRootLogin" /etc/ssh/sshd_config

# 4. Estableix permisos correctes al directori web
sudo find /var/www/misite -type f -exec chmod 644 {} \;
sudo find /var/www/misite -type d -exec chmod 755 {} \;
sudo chown -R www-data:www-data /var/www/misite
```
