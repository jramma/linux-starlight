## Exercici 6.1 — Gestió de paquets amb apt

### Context

`apt` és el gestor de paquets d'Ubuntu/Debian. Qualsevol programari del sistema operatiu —des de servidors web fins a eines de terminal— s'instal·la, actualitza i elimina amb apt.

### Tasca 1 — Cercar, instal·lar i eliminar paquets

```bash
# Actualitza la llista de paquets disponibles (no instal·la res)
sudo apt update

# Cerca informació sobre un paquet abans d'instal·lar-lo
apt show htop

# Instal·la htop (monitor de recursos millorat)
sudo apt install -y htop

# Prova'l
htop
# Prem Q per sortir
```

```bash
# Instal·la diverses eines alhora
sudo apt install -y tree curl wget

# Prova tree (mostra l'arbre de directoris)
tree /etc/apt/

# Elimina curl però conserva la seva configuració
sudo apt remove curl

# Elimina curl i la seva configuració (purge)
sudo apt purge curl

# Elimina paquets odis (instal·lats com a dependències i ja no necessaris)
sudo apt autoremove -y
```

### Tasca 2 — Instal·lar un paquet .deb manualment

A vegades un paquet no està als repositoris i l'has de descarregar com a `.deb`:

```bash
# Descarrega el paquet .deb de bat (un cat millorat amb sintaxi acolorida)
wget "https://github.com/sharkdp/bat/releases/download/v0.24.0/bat_0.24.0_amd64.deb" \
  -O /tmp/bat.deb

# Instal·la'l amb dpkg
sudo dpkg -i /tmp/bat.deb

# Prova'l
bat /etc/hosts
```

Si `dpkg` falla per dependències: `sudo apt install -f -y` i torna-ho a intentar.

## Exercici 6.2 — Script d'informe del sistema

### Context

Els sysadmins programen informes automàtics per detectar problemes abans que els usuaris els reportin. Crearàs un script que recull l'estat del servidor.

### Setup — una sola vegada

```bash
# Crea el directori on aniran els informes
mkdir -p ~/informes
```

### Tasca 1 — Crear l'script pas a pas

Primer entén cada peça abans de muntar-les juntes:

```bash
# Prova a la terminal cadascun d'aquests comandaments:

# 1. Data i hora actual
date '+%Y-%m-%d %H:%M:%S'

# 2. Hostname del servidor
hostname

# 3. Ús de disc de l'arrel
df -h / | tail -1

# 4. Memòria disponible
free -h | grep "Mem:"

# 5. Càrrega del sistema (últims 1, 5 i 15 minuts)
uptime | awk -F'load average:' '{print $2}'

# 6. Serveis caiguts (si no n'hi ha cap, no mostra res)
systemctl list-units --failed --no-legend 2>/dev/null | head -5
```

Ara crea l'script:

```bash
sudo nano /usr/local/bin/informe_sistema.sh
```

Escriu:

```bash
#!/bin/bash
# informe_sistema.sh — Informe diari de l'estat del servidor

LOGFILE="/var/log/informes/informe_$(date '+%Y-%m-%d').log"
SEPARADOR="========================================"

# Crea el directori si no existeix
mkdir -p /var/log/informes

{
  echo "$SEPARADOR"
  echo "INFORME DEL SERVIDOR — $(date '+%Y-%m-%d %H:%M:%S')"
  echo "$SEPARADOR"
  echo ""

  echo "## Servidor"
  echo "Hostname: $(hostname)"
  echo "Kernel:   $(uname -r)"
  echo ""

  echo "## Disc"
  df -h | grep -v tmpfs
  echo ""

  echo "## Memòria"
  free -h
  echo ""

  echo "## Càrrega del sistema"
  uptime
  echo ""

  echo "## Serveis caiguts"
  FALLIDES=$(systemctl list-units --failed --no-legend 2>/dev/null)
  if [ -z "$FALLIDES" ]; then
    echo "Tots els serveis funcionen correctament."
  else
    echo "$FALLIDES"
  fi
  echo ""

  echo "## Últims errors del sistema"
  journalctl -p err -n 5 --no-pager 2>/dev/null || echo "(sense errors recents)"
  echo ""

  echo "$SEPARADOR"
  echo "Fi de l'informe"
} >> "$LOGFILE"

echo "Informe desat a: $LOGFILE"
```

Dona-li permisos d'execució i prova'l:

```bash
sudo chmod +x /usr/local/bin/informe_sistema.sh

# Prova manual
sudo /usr/local/bin/informe_sistema.sh

# Llegeix l'informe generat
cat /var/log/informes/informe_$(date '+%Y-%m-%d').log
```

L'informe ha de tenir totes les seccions emplenades. Si alguna està buida, revisa el comandament corresponent.

## Exercici 6.3 — Programar con systemd timer

### Tasca 1 — Crear el servei

```bash
sudo nano /etc/systemd/system/informe-sistema.service
```

```ini
[Unit]
Description=Informe diari de l'estat del servidor
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/informe_sistema.sh

[Install]
WantedBy=multi-user.target
```

### Tasca 2 — Crear el timer

```bash
sudo nano /etc/systemd/system/informe-sistema.timer
```

```ini
[Unit]
Description=Timer per a informe diari a les 7:00

[Timer]
OnCalendar=*-*-* 07:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

<Aside type="note" title="Què fa Persistent=true?">
Si el servidor estava apagat a les 7:00, en engegar-se executarà l'informe que s'ha perdut. Sense aquesta opció, l'execució es perdria.
</Aside>

### Tasca 3 — Activar i provar

```bash
sudo systemctl daemon-reload

# Habilita el timer
sudo systemctl enable informe-sistema.timer

# Arrenca el timer
sudo systemctl start informe-sistema.timer

# Verifica l'estat
systemctl status informe-sistema.timer

# Verifica quan és la propera execució
systemctl list-timers | grep informe

# Força una execució immediata per provar
sudo systemctl start informe-sistema.service

# Verifica als logs de systemd
journalctl -u informe-sistema.service -n 5 --no-pager

# Verifica que s'ha generat l'informe
ls -la /var/log/informes/
cat /var/log/informes/informe_$(date '+%Y-%m-%d').log
```
